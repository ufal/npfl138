# Generative Models

![w=46%,f=right](generative_models.png)

There are several approaches how to represent a probability distribution
$P(‚Āá‚Üíx)$.
~~~
**Likelihood-based models** represent the probability density function directly,
often using an unnormalized probabilistic model (also called energy-based model;
i.e., specifying a non-zero _score_ or _density_ or _logits_):
$$P_{‚Üíőł}(‚Āá‚Üíx) = \frac{e^{f_{‚Üíőł}(‚Āá‚Üíx)}}{Z_{‚Üíőł}}.$$

~~~
However, estimating the normalization constant $Z_{‚Üíőł} = ‚ąęe^{f_{‚Üíőł}(‚Āá‚Üíx)}\d‚Āá‚Üíx$ is
often intractable.

~~~
- We can compute $Z_{‚Üíőł}$ by restricting the model architecture (sequence
  modeling, invertible networks in normalizing flows);
~~~
- we can only approximate it (using for example variational inference as in
  VAE);
~~~
- we can use **implicit generative models**, which avoid representing likelihood
  (like GANs).

# Diffusion Models ‚Äď Diffusion Process, Reverse Process

![w=80%,h=center](ddpm_model.svgz)

Given a data point $‚Āá‚Üíx_0$ from a real data distribution $q(‚Āá‚Üíx)$, we define
a $T$-step _diffusion process_ (or the _forward process_) which gradually adds
Gaussian noise to the input image:

$$q(‚Āá‚Üíx_{1:T}|‚Āá‚Üíx_0) = ‚ąŹ\nolimits_{t=1}^\T q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1}).$$

~~~
Our goal is to reverse the forward process $q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1})$, i.e., generate an
image by starting with $‚Āá‚Üíx_T ‚ąľ ūĚďĚ(‚Üí0, ‚áČI)$ and then performing the forward
process in reverse.
~~~
We therefore learn a model $p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)$ to approximate the reverse
of $q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1})$, and obtain a _reverse process_:
$$p_{‚Üíőł}(‚Āá‚Üíx_{0:T}) = p(‚Āá‚Üíx_T) ‚ąŹ\nolimits_{t=1}^\T p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t).$$

# Diffusion Models ‚Äď Model Overview

![w=88%,mw=19%,h=right,f=right](imagen_architecture_overall.svgz)

The $p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)$ is commonly modelled using a U-Net architecture
with skip connections.

~~~
### Training

During training, we randomly sample a time step $t$, and perform an update
of the parameters $‚Üíőł$ in order for $p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)$ to
better approximate the reverse of $q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1})$.

~~~
### Sampling

In order to sample an image, we start by sampling $‚Āá‚Üíx_T ‚ąľ ūĚďĚ(‚Üí0, ‚áČI)$,
and then perform $T$ steps of the reverse process by sampling
$‚Āá‚Üíx_{t-1} ‚ąľ p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)$ for $t$ from $T$ down to 1.

# Normal Distribution Reminder

![w=28%,f=right](../01/normal_distribution.svgz)

Normal (or Gaussian) distribution is a continuous distribution parametrized by
a mean $őľ$ and variance $ŌÉ^2$:

$$ūĚďĚ(x; őľ, ŌÉ^2) = \sqrt{\frac{1}{2ŌÄŌÉ^2}} \exp \left(-\frac{(x - őľ)^2}{2ŌÉ^2}\right)$$

~~~
For a $D$-dimensional vector $‚Üíx$, the multivariate Gaussian distribution takes
the form
$$ūĚďĚ(‚Üíx; ‚Üíőľ, ‚áČő£) ‚ČĚ \frac{1}{\sqrt{(2ŌÄ)^D |‚áČő£|}} \exp \left(-\frac{1}{2}(‚Üíx-‚Üíőľ)^\T ‚áČő£^{-1} (‚Üíx-‚Üíőľ) \right).$$

~~~
The biggest difference compared to the single-dimensional case is the _covariance
matrix_ $‚áČő£$, which is (in the non-degenerate case, which is the only one
considered here) a _symmetric positive-definite matrix_ of size $D √ó D$.

~~~
However, in this lecture we will only consider _isotropic_ distribution, where $‚áČő£ = ŌÉ^2‚áČI$:
$$ūĚďĚ(‚Üíx; ‚Üíőľ, ŌÉ^2‚áČI) = ‚ąŹ\nolimits_i ūĚďĚ(x_i; őľ_i, ŌÉ^2).$$

---
# Normal Distribution Reminder

- A normally-distributed random variable $‚Āá‚Üíx ‚ąľ ūĚďĚ(‚Üíőľ, ŌÉ^2‚áČI)$ can be written using
  the reparametrization trick also as
  $$‚Āá‚Üíx = ‚Üíőľ + ŌÉ ‚Āá‚Üíe,\textrm{~~where~~}‚Āá‚Üíe ‚ąľ ūĚďĚ(‚Üí0, ‚áČI).$$

~~~
- The sum of two independent normally-distributed random variables $‚Āá‚Üíx_1 ‚ąľ ūĚďĚ(‚Üíőľ_1, ŌÉ_1^2‚áČI)$
  and $‚Āá‚Üíx_2 ‚ąľ ūĚďĚ(‚Üíőľ_2, ŌÉ_2^2‚áČI)$ has normal distribution $‚ĀáūĚďĚ\big(‚Üíőľ_1 + ‚Üíőľ_2, (ŌÉ_1^2 + ŌÉ_2^2)‚áČI\big)$.

~~~
  Therefore, if we have two standard normal random variables $‚Āá‚Üíe_1, ‚Āá‚Üíe_2 ‚ąľ ūĚďĚ(‚Üí0, ‚áČI)$, then
  $$ŌÉ_1 ‚Āá‚Üíe_1 + ŌÉ_2 ‚Āá‚Üíe_2 = \sqrt{ŌÉ_1^2 + ŌÉ_2^2} ‚Āá‚Üíe$$
  for a standard normal random variable $‚Āá‚Üíe ‚ąľ ūĚďĚ(‚Üí0, ‚áČI)$.

# DDPM ‚Äď The Forward Process

We now describe _Denoising Diffusion Probabilistic Models (DDPM)_.

~~~
![w=80%,h=center](ddpm_model.svgz)

Given a data point $‚Āá‚Üíx_0$ from a real data distribution $q(‚Āá‚Üíx)$, we define
a $T$-step _diffusion process_ (or the _forward process_) which gradually adds
Gaussian noise according to some variance schedule $ő≤_1, ‚Ä¶, ő≤_T$:

$\displaystyle \kern11em{}\mathllap{q(‚Āá‚Üíx_{1:T}|‚Āá‚Üíx_0)} = ‚ąŹ_{t=1}^\T q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1}),$

~~~
$\displaystyle \kern11em{}\mathllap{q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1})} = ūĚďĚ(‚Āá‚Üíx_t; \sqrt{1 - ő≤_t} ‚Āá‚Üíx_{t-1}, ő≤_t ‚áČI),$

~~~
$\displaystyle \kern11em{}\mathllap{} = \sqrt{1 - ő≤_t} ‚Āá‚Üíx_{t-1} + \sqrt{ő≤_t} ‚Āá‚Üíe\textrm{~~for~~}‚Āá‚Üíe‚ąľūĚďĚ(‚Üí0, ‚áČI).$

~~~
More noise gets gradually added to the original image $‚Āá‚Üíx_0$, converging to
pure Gaussian noise.

---
# DDPM ‚Äď The Forward Process

Let $őĪ_t = 1-ő≤_t$ and $őĪŐĄ_t = ‚ąŹ_{i=1}^t őĪ_i$.
~~~
Then we have

$\displaystyle\kern3em{}\mathllap{‚Āá‚Üíx_t} = \sqrt{őĪ_t} \textcolor{blue}{‚Āá‚Üíx_{t-1}} + \sqrt{1-őĪ_t}‚Āá‚Üíe_t$

~~~
$\displaystyle\kern3em{} = \sqrt{őĪ_t} \textcolor{blue}{\big(\sqrt{őĪ_{t-1}} ‚Āá‚Üíx_{t-2} + \sqrt{1-őĪ_{t-1}}‚Āá‚Üíe_{t-1}\big)} + \sqrt{1-őĪ_t}‚Āá‚Üíe_t$

~~~
$\displaystyle\kern3em{} = \sqrt{őĪ_t őĪ_{t-1}} ‚Āá‚Üíx_{t-2} + \sqrt{őĪ_t(1-őĪ_{t-1}) + (1-őĪ_t)}‚Āá‚ÜíeŐĄ_{t-1}$

~~~
$\displaystyle\kern3em{} = \sqrt{őĪ_t őĪ_{t-1}} ‚Āá‚Üíx_{t-2} + \sqrt{1 - őĪ_t őĪ_{t-1}}‚Āá‚ÜíeŐĄ_{t-1}$

~~~
$\displaystyle\kern3em{} = \sqrt{őĪ_t őĪ_{t-1} őĪ_{t-2}} ‚Āá‚Üíx_{t-3} + \sqrt{1 - őĪ_t őĪ_{t-1} őĪ_{t-2}}‚Āá‚ÜíeŐĄ_{t-2}$

~~~
$\displaystyle\kern3em{} = ‚Ä¶$

~~~
$\displaystyle\kern3em{} = \sqrt{őĪŐĄ_t} ‚Āá‚Üíx_0 + \sqrt{1-őĪŐĄ_t}‚Āá‚ÜíeŐĄ_0$

for standard normal random variables $‚Āá‚Üíe_i$ and $‚Āá‚ÜíeŐĄ_i$.

~~~
In other words, we have shown that $q(‚Āá‚Üíx_t | ‚Āá‚Üíx_0) = ūĚďĚ\big(\sqrt{őĪŐĄ_t}‚Āá‚Üíx_0, (1-őĪŐĄ_t)‚áČI\big)$.

~~~
Therefore, if $őĪŐĄ_t ‚Üí 0$ as $t ‚Üí ‚ąě$, the $‚Āá‚Üíx_t$ converges to $ūĚďĚ(‚Üí0, ‚áČI)$ as $t ‚Üí ‚ąě$.

# DDPM ‚Äď Noise Schedule

![w=50%,f=right](diffusion_schedule.svgz)

Originally, linearly increasing sequence of noise variances
$ő≤_1=0.0001, ‚Ä¶, ő≤_T=0.04$ was used.

~~~
However, the resulting sequence $őĪŐĄ_t$ was not ideal (nearly the whole second
half of the diffusion process was mostly just random noise), so later a cosine
schedule was proposed:
$$őĪŐĄ_t = \frac{1}{2}\Big(\cos(t/T ‚čÖ ŌÄ)+1\Big).$$

~~~
In practice, we want to avoid both the values of 0 and 1, and keep $őĪ_t$ in $[őĶ, 1-őĶ]$ range.

---
# DDPM ‚Äď Noise Schedule

We assume the images $‚Āá‚Üíx_0$ have zero mean and unit variance (we normalize them
to achieve that).
~~~
Then every
$$q(‚Āá‚Üíx_t|‚Āá‚Üíx_0) = \textcolor{red}{\sqrt{őĪŐĄ_t}} ‚Āá‚Üíx_0 + \textcolor{blue}{\sqrt{1-őĪŐĄ_t}}‚Āá‚Üíe$$
has also zero mean and unit variance.

~~~
![w=29%,f=right](signal_noise_rates.png)

The $\textcolor{red}{\sqrt{őĪŐĄ_t}}$ and $\textcolor{blue}{\sqrt{1-őĪŐĄ_t}}$ can be
considered as the _signal rate_ and the _noise rate_.

~~~
Because $\textcolor{red}{\sqrt{őĪŐĄ_t}}^2 + \textcolor{blue}{\sqrt{1-őĪŐĄ_t}}^2 = 1$,
the signal rate and the noise rate form a circular arc. The proposed cosine
schedule
$$\begin{aligned}
  \textcolor{red}{\sqrt{őĪŐĄ_t}} &= \cos(t/T ‚čÖ ŌÄ/2), \\
  \textcolor{blue}{\sqrt{1-őĪŐĄ_t}} &= \sin(t/T ‚čÖ ŌÄ/2),
\end{aligned}$$
corresponds to an uniform movement on this arc.

# DDPM ‚Äď The Reverse Process

![w=80%,h=center](ddpm_model.svgz)

In order to be able to generate images, we therefore learn a model
$p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)$ to approximate the reverse of $q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1})$.

~~~
When $ő≤_t$ is small, this reverse is nearly Gaussian, so we represent $p_{‚Üíőł}$
as
$$p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t) = ūĚďĚ\big(‚Āá‚Üíx_{t-1}; ‚Üíőľ_{‚Üíőł}(‚Āá‚Üíx_t, t), ŌÉ_t^2‚áČI\big)$$
for some fixed sequence of $ŌÉ_1, ‚Ä¶, ŌÉ_T$.

~~~
The whole reverse process is then
$$p_{‚Üíőł}(‚Āá‚Üíx_{0:T}) = p(‚Āá‚Üíx_T) ‚ąŹ\nolimits_{t=1}^\T p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t).$$

---
# DDPM ‚Äď Loss

We now want to derive the loss. First note that the reverse of $q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1})$
is actually tractable when conditioning on $‚Āá‚Üíx_0$:


$\displaystyle\kern9em{}\mathllap{q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t, ‚Āá‚Üíx_0)} = ūĚďĚ\big(‚Āá‚Üíx_{t-1}; \textcolor{blue}{‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0)}, \textcolor{green}{ő≤ŐÉ_t}‚áČI\big),$

~~~
$\displaystyle\kern9em{}\mathllap{\textcolor{blue}{‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0)}} = \frac{\sqrt{őĪŐĄ_{t-1}}ő≤_t}{1-őĪŐĄ_t}‚Āá‚Üíx_0 + \frac{\sqrt{őĪ_t}(1-őĪŐĄ_{t-1})}{1-őĪŐĄ_t}‚Āá‚Üíx_t,$

~~~
$\displaystyle\kern9em{}\mathllap{\textcolor{green}{ő≤ŐÉ_t}} = \frac{1-őĪŐĄ_{t-1}}{1-őĪŐĄ_t}ő≤_t.$

~~~
We present the proof on the next slide for completeness.

Starting with the Bayes' rule, we get

$\displaystyle\kern7em{}\mathllap{q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t, ‚Āá‚Üíx_0)} = \textcolor{purple}{q(‚Āá‚Üíx_t | ‚Āá‚Üíx_{t-1}, ‚Āá‚Üíx_0)} \frac{\textcolor{darkcyan}{q(‚Āá‚Üíx_{t-1} | ‚Āá‚Üíx_0)}}{\textcolor{red}{q(‚Āá‚Üíx_t | ‚Āá‚Üíx_0)}}$

# DDPM ‚Äď Loss

The full derivation of the loss is available in the Bonus Content of this
presentation. The resulting loss is

$$L_t = ūĚĒľ\bigg[\frac{1}{2\|ŌÉ_t‚áČI\|^2}\Big\| ‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0) - ‚Üíőľ_{‚Üíőł}(‚Āá‚Üíx_t, t) \Big\|^2\bigg].$$

Denoting $‚Āá‚Üíx_t = \sqrt{őĪŐĄ_t} ‚Āá‚Üíx_0 + \sqrt{1-őĪŐĄ_t}‚Āá‚Üíe_t$, it is possible to
change the model to predict $‚ÜíőĶ_{‚Üíőł}(‚Āá‚Üíx_t, t) = (‚Üíx_t - \sqrt{őĪŐĄ_t} ‚Āá‚Üíx_0) / \sqrt{1-őĪŐĄ_t}$ instead of $‚Üíőľ_{‚Üíőł}(‚Āá‚Üíx_t, t)$.
~~~
The loss then becomes

$$L_t = ūĚĒľ\bigg[\frac{(1-őĪ_t)^2}{2őĪ_t(1-őĪŐĄ_t)\|ŌÉ_t‚áČI\|^2}\Big\| ‚Āá‚Üíe_t - ‚ÜíőĶ_{‚Üíőł}\big(\sqrt{ŠĺĪ_t} ‚Āá‚Üíx_0 + \sqrt{1-ŠĺĪ_t}‚Āá‚Üíe_t, t\big) \Big\|^2\bigg].$$

~~~
The authors found that training without the weighting term performs better, so
the final loss is
$$L_t^\textrm{simple} = ūĚĒľ_{t‚ąą\{1..T\},‚Āá‚Üíx_0,‚Āá‚Üíe_t}\Big[\big\| ‚Āá‚Üíe_t - ‚ÜíőĶ_{‚Üíőł}\big(\sqrt{ŠĺĪ_t} ‚Āá‚Üíx_0 + \sqrt{1-ŠĺĪ_t}‚Āá‚Üíe_t, t\big) \big\|^2\Big].$$

~~~
Note that both losses have the same optimum if we used independent $‚ÜíőĶ_{‚Üíőł_t}$ for every $t$.

# Flow Matching

In the past years (since 2022), the dominant approach for generating images has
been based on **diffusion models** (used by Stable Diffusion, DALL-E, ‚Ä¶).

![w=43%,h=center](../12/stable_diffusion.jpg)

~~~
The diffusion models are deeply connected to **score-based generative models**,
which were developed independently, but are just a different perspective on the
same model family.

~~~
Recently, **conditional flow matching** was proposed as a generalization of both
these approaches, and that is the method we now describe in detail.

---
# Generative Modeling

The general framework of generative modeling assumes we have samples $‚Üíx^{(1)},
‚Üíx^{(2)}, ‚Ä¶, ‚Üíx^{(N)}$ from the data generating distribution $p_\textrm{data}$,
and the main challenges we should overcome are:
~~~
- provide fast sampling,
~~~
- generate high-quality samples,
~~~
- properly cover the density of $p_\textrm{data}$.

~~~ ~~~~
- provide fast sampling (diffusion models were originally not great here),
- generate high-quality samples (VAE struggles with this goal),
- properly cover the density of $p_\textrm{data}$ (the main issue of GANs).

~~~
Modern approach to generative modeling is to start with a simple base
distribution $p_0$, usually a standard Gaussian $ūĚďĚ(‚Üí0, ‚áČI)$, and learn
a mapping that transforms that distribution into $p_\textrm{data}$.

![w=60%,h=center](pushforward.svgz)

~~~
Sampling then can be performed by sampling from $p_0$ and performing the
transformation.

# Flow Matching

![w=54%,h=center](flow_visualization.png)

The important concepts used by flow matching are:

~~~
- the **probability density path** $p \colon [0, 1] √ó ‚ĄĚ^d ‚Üí ‚ĄĚ_{>0}$, which is
  a time-dependent probability density function, i.e., $‚ąę p_t(‚Üíx) \d ‚Üíx = 1$;
~~~

  - this probability density path should transform the prior $p_0$ into
    $p_1=p_\textrm{data}$
~~~
- a **time-dependent vector field** $u \colon [0, 1] √ó ‚ĄĚ^d ‚Üí ‚ĄĚ^d$, which can be
  used to construct a **flow** $ŌÜ \colon [0, 1] √ó ‚ĄĚ^d ‚Üí ‚ĄĚ^d$ via an ordinary differential
  equation:
  $$\tfrac{d}{dt} ŌÜ_t(‚Üíx) = u_t(ŌÜ_t(‚Üíx)),~~~~ŌÜ_0(‚Üíx) = ‚Üíx.$$

---
style: .katex-display { margin: .7em 0 }
# Flow Matching

Note that the solution of the flow ODE
$$\tfrac{d}{dt} ŌÜ_t(‚Üíx) = u_t(ŌÜ_t(‚Üíx)),~~~~ŌÜ_0(‚Üíx) = ‚Üíx$$
is unique when $u_t$ is Lipschitz continuous in $‚Üíx$ and
continuous in $t$ (Picard‚ÄďLindel√∂f theorem).

~~~
![w=34%,f=right](resnet_and_ode.svgz)

Recall that in a residual network, we update the current value by adding the
result of a residual block
$$‚Üíh_{t+1} = ‚Üíh_t + f(‚Üíh_t; ‚Üíőł_t),$$
~~~
which we can also write as
$$‚Üíh_{t+1} - ‚Üíh_t = f(‚Üíh_t; ‚Üíőł_t),$$
where we can interpret the residual block as a ‚Äúdiscrete derivative‚ÄĚ.

~~~
Therefore, the flow can be considered to be a continuous generalization of
residual networks.

---
# Flow Matching: Transport Equation

A vector field $u_t$ is said to **generate** a probability density path $p_t$
if the _transport equation_ holds:
$$\tfrac{d}{dt} p_t(‚Üíx) = -\operatorname{div}\big(p_t(‚Üíx) u_t(‚Üíx)\big),$$

~~~
![w=90%,h=right,mw=40%,f=right](divergence.svgz)

where the **divergence** $\operatorname{div}(‚Üíz) ‚ČĚ ‚ąĎ_i \tfrac{‚ąā z_i}{‚ąāx_i}$
is a vector operator that operates on a vector field, producing for every
point a scalar value, the field's _source_ at that point (a positive
value means a point is a source; negative if it is a sink).

~~~
The $p_t(‚Üíx) u_t(‚Üíx)$ is a _flux_, the probability mass passing through
every point of the space (in the direction of the vector field).

---
style: .katex-display { margin: .5em 0 }
# Flow Matching: Basic Objective

Assume we have a base distribution $p_0$, usually $ūĚďĚ(‚Üí0, ‚áČI)$, and samples
$‚Üíx^{(i)}$ of the data generating distribution $p_1 = p_\textrm{data}$.
~~~
Given a target probability density path $p_t$ and a corresponding vector
field $u_t$ generating $p_t$, the _flow matching (FM) objective_ is
$$ūĚďõ_\textrm{FM}(‚Üíőł) ‚ČĚ ūĚĒľ_{t, ‚Üíx‚ąľp_t(‚Üíx)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx)\|^2.$$

~~~
However, we need to overcome that we have no prior knowledge on how the $p_t$
and $u_t$ should look like given that there are many possible probability
density paths $p_t$, and that for an arbitrary $p_t$, we usually do not have
access to the closed form of its generating vector field $u_t$.

# Flow Matching: Basic Overview

Once we solve the remaining issues with the flow matching objective,
we train a NN model $v_t(‚Üíx; ‚Üíőł)$ by predicting (matching) the vector field
$u_t(‚Üíx)$.
~~~
Usually, the U-Net architecture with skip connections is used to model $v_t$.

~~~
### Training

During training, we minimize an objective corresponding to flow matching
$$ūĚďõ_\textrm{FM}(‚Üíőł) ‚ČĚ ūĚĒľ_{t, ‚Üíx‚ąľp_t(‚Üíx)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx)\|^2,$$
i.e., by performing a regression on the vector predicted by the model $v_t$.

~~~
### Sampling

In order to generate an image, we start by sampling $‚Āá‚Üíx_0 ‚ąľ p_0$, and then
perform numerical integration by the Euler method using $T$ steps by
$$‚Üíx_{t+\frac{1}{T}} ‚Üź ‚Üíx_t + \tfrac{1}{T} v_t(‚Üíx_t; ‚Üíőł).$$

~~~
More involved methods like the midpoint or Runge-Kutta can also be used.

# Conditional and Marginal Probability Paths

Instead of directly defining the probability density path, we can construct it
as a mixture of simpler probability paths. Together with the fact that
we do not have direct access to the data generating distribution $p_1
= p_\textrm{data}$ apart from its samples, we turn to **conditional
probability paths** $p_t(‚Üíx | ‚Üíx_1)$, which we design so that
~~~
- $p_0(‚Üíx | ‚Üíx_t) = p_0(‚Üíx)$,
~~~
- $p_1(‚Üíx | ‚Üíx_1)$ is tightly concentrated around $‚Üíx_1$, for example by using
  a normal distribution with a small variance $ŌÉ^2_\textrm{min}>0$:
  $$p_1(‚Üíx | ‚Üíx_1) = ūĚďĚ(‚Üíx | ‚Üíx_1, ŌÉ^2_\textrm{min}‚áČI).$$

~~~
We can then define the _marginal probability path_ as
$$p_t(‚Üíx) ‚ČĚ ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}} \big[p_t(‚Üíx | ‚Üíx_1)\big] \textcolor{gray}{= ‚ąęp_t(‚Üíx | ‚Üíx_1) p_\textrm{data}(‚Üíx_1) \d ‚Üíx_1}.$$

~~~
Because we defined the conditional probability paths to concentrate tightly
around $‚Üíx_1$, the marginal probability $p_1(‚Üíx) ‚Čą p_\textrm{data}$.

---
style: .katex-display { margin: .7em 0 }
# Conditional and Marginal Vector Fields

Analogously to how we defined the marginal probability path using the conditional
probability paths, it is also possible to define the _marginal vector field_
(the vector field of the marginal probability path) using the _conditional
vector fields_ $u_t(‚Üíx | ‚Üíx_1)$ (the vector fields of the conditional
probability paths):

$$u_t(‚Üíx) ‚ČĚ ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}} \bigg[u_t(‚Üíx | ‚Üíx_1) \frac{p_t(‚Üíx | ‚Üíx_1)}{p_t(‚Üíx)}\bigg] \textcolor{gray}{= ‚ąęu_t(‚Üíx | ‚Üíx_1) \frac{p_t(‚Üíx | ‚Üíx_1)p_\textrm{data}(‚Üíx)}{p_t(‚Üíx)} \d ‚Üíx_1}.$$

~~~
Such a marginal vector field actually generates the marginal probability path.

~~~
![w=70%,h=center,mw=50%](g2g_vector_field_1.png)![w=70%,h=center,mw=50%](g2g_vector_field_2.png)

---
style: .katex-display { margin: .7em 0 }
# Conditional and Marginal Vector Fields

To verify that the marginal vector field generates the marginal probability
path, we need to check that $p_t$ and $u_t$ satisfy the transport equation:

$\displaystyle \kern7em{}\mathllap{\tfrac{d}{dt} p_t(‚Üíx)} = ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}}\Big[\tfrac{d}{dt} p_t(‚Üíx | ‚Üíx_1)\Big]$

~~~
$\displaystyle \kern7em{} = ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}}\Big[-\operatorname{div}\big(u_t(‚Üíx | ‚Üíx_1) p_t(‚Üíx | ‚Üíx_1)\big)\Big]$

~~~
$\displaystyle \kern7em{} = -\operatorname{div}\Big(ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}}\big[u_t(‚Üíx | ‚Üíx_1) p_t(‚Üíx | ‚Üíx_1)\big]\Big)$

~~~
$\displaystyle \kern7em{} = -\operatorname{div}\Big(\underbrace{ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}}\big[u_t(‚Üíx | ‚Üíx_1) p_t(‚Üíx | ‚Üíx_1)\textcolor{blue}{/p_t(‚Üíx)}\big]}_{\textrm{definition~of~}u_t(‚Üíx)}\textcolor{blue}{p_t(‚Üíx)}\Big)$

~~~
$\displaystyle \kern7em{} = -\operatorname{div}\big(u_t(‚Üíx)p_t(‚Üíx)\big).$

~~~
Note that swapping a derivative and an integral requires various smoothness
conditions; we assume all our objects ‚Äúnice enough‚ÄĚ.

---
# Conditional Flow Matching

Even with the definition of the marginal vector field, obtaining an unbiased
estimate of the flow matching objective

$$ūĚďõ_\textrm{FM}(‚Üíőł) ‚ČĚ ūĚĒľ_{t, ‚Üíx‚ąľp_t(‚Üíx)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx)\|^2$$

is still intractable (the integrals in
the definition of the conditional probability path/vector field are
intractable).

~~~
However, we can use the following simplified _conditional flow matching (CFM)
objective_:

$$ūĚďõ_\textrm{CFM}(‚Üíőł) ‚ČĚ ūĚĒľ_{t, ‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx ‚ąľ p_t(‚Üíx|‚Üíx_1)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx | ‚Üíx_1)\|^2.$$

~~~
This objective allows unbiased estimates, given that only the conditional variants
of $p_t$ and $u_t$ are needed.

~~~
Importantly, while this objective $ūĚďõ_\textrm{CFM}$ is different from the
original flow matching objective $ūĚďõ_\textrm{FM}$, it has the same gradients with
respect to $‚Üíőł$.

---

style: .katex-display { margin: .65em 0 }
# Conditional Flow Matching

$$\begin{aligned}
  ūĚďõ_\textrm{FM}(‚Üíőł) &‚ČĚ ūĚĒľ_{t,‚Üíx‚ąľp_t(‚Üíx)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx)\|^2 \\
  ūĚďõ_\textrm{CFM}(‚Üíőł) &‚ČĚ ūĚĒľ_{t, ‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx ‚ąľ p_t(‚Üíx|‚Üíx_1)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx | ‚Üíx_1)\|^2 \\
\end{aligned}$$

To show that the gradients $‚ąá_{‚Üíőł}$ of $ūĚďõ_\textrm{FM}$ and $ūĚďõ_\textrm{CFM}$ are
the same, we start with:

~~~
$$\begin{aligned}
  \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx)\|^2 &= (v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx))^\T(v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx)) \\
                              &= \|v_t(‚Üíx; ‚Üíőł)\|^2 - 2 v_t(‚Üíx; ‚Üíőł)^\T u_t(‚Üíx) + \|u_t(‚Üíx)\|^2 \\
  \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx | ‚Üíx_1)\|^2 &= \|v_t(‚Üíx; ‚Üíőł)\|^2 - 2 v_t(‚Üíx; ‚Üíőł)^\T u_t(‚Üíx | ‚Üíx_1) + \|u_t(‚Üíx | ‚Üíx_1)\|^2
\end{aligned}$$

~~~
First, we note that $u_t$ is independent of $‚Üíőł$.
~~~
Second, because
$$p_t(‚Üíx) ‚ČĚ ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}} \big[p_t(‚Üíx | ‚Üíx_1)\big],$$
~~~
we get that
$$ūĚĒľ_{‚Üíx‚ąľp_t(‚Üíx)} \|v_t(‚Üíx; ‚Üíőł)\|^2 = ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx ‚ąľ p_t(‚Üíx|‚Üíx_1)} \|v_t(‚Üíx; ‚Üíőł)\|^2.$$

---
class: dbend
style: .katex-display { margin: .65em 0 }
# Conditional Flow Matching

$$\begin{aligned}
  ūĚďõ_\textrm{FM}(‚Üíőł) &‚ČĚ ūĚĒľ_{t, ‚Üíx‚ąľp_t(‚Üíx)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx)\|^2 \\
  ūĚďõ_\textrm{CFM}(‚Üíőł) &‚ČĚ ūĚĒľ_{t, ‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx ‚ąľ p_t(‚Üíx|‚Üíx_1)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx | ‚Üíx_1)\|^2 \\
\end{aligned}$$

Finally, to handle the middle (dot-product) term from the expansion of the squares,
using

$$u_t(‚Üíx) ‚ČĚ ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}} \bigg[u_t(‚Üíx | ‚Üíx_1) \frac{p_t(‚Üíx | ‚Üíx_1)}{p_t(‚Üíx)}\bigg]$$

~~~
we get that
$$\begin{aligned}
  ūĚĒľ_{‚Üíx‚ąľp_t(‚Üíx)} \big[v_t(‚Üíx; ‚Üíőł)^\T u_t(‚Üíx)\big] &= ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data},‚Üíx‚ąľp_t(‚Üíx)} \big[v_t(‚Üíx; ‚Üíőł)^\T \big(u_t(‚Üíx | ‚Üíx_1) p_t(‚Üíx | ‚Üíx_1) / p_t(‚Üíx)\big) \big] \\
                                               &= ūĚĒľ_{‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx ‚ąľ p_t(‚Üíx|‚Üíx_1)} \big[v_t(‚Üíx; ‚Üíőł)^\T u_t(‚Üíx | ‚Üíx_1) \big]. \\
\end{aligned}$$

---
# Construction of the Conditional Probability Paths

We now define the conditional probability paths that we will use. We consider
normal distributions with time-dependent mean $őľ_t(‚Üíx_1)$ and variance
$ŌÉ^2_t(‚Üíx_1)$:

$$p_t(‚Üíx | ‚Üíx_1) ‚ČĚ ūĚďĚ\big(‚Üíx | őľ_t(‚Üíx_1), ŌÉ^2_t(‚Üíx_1)‚áČI\big).$$

~~~
We require
- $őľ_0(‚Üíx_1) = 0$ and $ŌÉ^2_0(‚Üíx_1) = 1$ so that $p_0(‚Üíx | ‚Üíx_1) = p_0(‚Üíx)$, and

~~~
- $őľ_t(‚Üíx_1) = ‚Üíx_1$ and $ŌÉ^2_1(‚Üíx_1) = ŌÉ^2_\textrm{min}$ so that $p_1(‚Üíx
  | ‚Üíx_1)$ is concentrated around $‚Üíx_1$.

~~~
While there are infinitely many vector fields generating these probability
paths, we use the simplest one, corresponding to the flow (dependent on $‚Üíx_1$)
$$ŌÜ_t(‚Üíx) = ŌÉ_t(‚Üíx_1) ‚Üíx + őľ_t(‚Üíx_1),$$

which is an affine transformation mapping standard normal distribution to
a normal distribution with mean $őľ_t(‚Üíx_1)$ and variance $ŌÉ^2_t(‚Üíx_1)$.

---
style: .katex-display { margin: .9em 0 }
# Construction of the Conditional Vector Field

We now derive the conditional vector field $u_t(‚Üíx | ‚Üíx_1)$ so that its flow is the
defined
$$ŌÜ_t(‚Üíx) = ŌÉ_t(‚Üíx_1) ‚Üíx + őľ_t(‚Üíx_1).$$

~~~
Recalling the flow ODE $\tfrac{d}{dt} ŌÜ_t(‚Üíx) = u_t(ŌÜ_t(‚Üíx))$, it could be used
to defined $u_t$, but unfortunately for $ŌÜ_t(‚Üíx)$, not for arbitrary $‚Üíx$.

~~~
However, the affine map $ŌÜ_t$ has an analytical inverse (assuming $ŌÉ_t(‚Üíx_1) > 0$)
$$\textcolor{blue}{ŌÜ^{-1}_t(‚Üíz)} = \textcolor{blue}{\frac{‚Üíz - őľ_t(‚Üíx_1)}{ŌÉ_t(‚Üíx_1)}}.$$

~~~
Therefore, when we consider $‚Üíz = ŌÜ_t(‚Üíx)$, we get $u_t(‚Üíz | ‚Üíx_1) = \textcolor{darkred}{ŌÜ'_t(\textcolor{blue}{ŌÜ^{-1}_t(‚Üíz)})}$.

~~~
Using the derivative $\textcolor{darkred}{ŌÜ'_t(‚Üíx | ‚Üíx_1)} = \textcolor{darkred}{ŌÉ'_t(‚Üíx_1) ‚Üíx + őľ'_t(‚Üíx_1)}$,
we obtain that

~~~
$$u_t(‚Üíz | ‚Üíx_1) = \frac{\textcolor{darkred}{ŌÉ'_t(‚Üíx_1)}}{\textcolor{blue}{ŌÉ_t(‚Üíx_1)}}\big(\textcolor{blue}{‚Üíz - őľ_t(‚Üíx_1)}\big) + \textcolor{darkred}{őľ'_t(‚Üíx_1)}.$$

---
# Construction of the Conditional Vector Field

The authors propose to use conditional paths with mean and variance changing
linearly in $t$:
$$őľ_t(‚Üíx_1) ‚ČĚ t‚Üíx_1,~~~ŌÉ_t(‚Üíx_1) ‚ČĚ 1 - (1-ŌÉ_\textrm{min})t.$$

~~~
Therefore, the corresponding flow and vector field are
$$\begin{gathered}
  ŌÜ_t(‚Üíx) = (1 - (1-ŌÉ_\textrm{min})t) ‚Üíx + t‚Üíx_1,\textcolor{gray}{\small~~~~such~a~flow~is~called~Optimal~Transport\kern-5em{}}\\
  u_t(‚Üíx | ‚Üíx_1) = \frac{-(1-ŌÉ_\textrm{min})}{1 - (1-ŌÉ_\textrm{min})t}\big(‚Üíx - t‚Üíx_1\big) + ‚Üíx_1 = \frac{‚Üíx_1-(1-ŌÉ_\textrm{min})‚Üíx}{1 - (1-ŌÉ_\textrm{min})t}.
\end{gathered}$$

~~~
Finally, recalling the general form $ūĚďõ_\textrm{CFM}(‚Üíőł) ‚ČĚ ūĚĒľ_{t, ‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx ‚ąľ p_t(‚Üíx|‚Üíx_1)} \|v_t(‚Üíx; ‚Üíőł) - u_t(‚Üíx | ‚Üíx_1)\|^2$,
~~~
for our specific case of OT flow we obtain
$$ūĚďõ_\textrm{CFM}(‚Üíőł) ‚ČĚ ūĚĒľ_{t, ‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx_0 ‚ąľ p_0} \big\|v_t\big(ŌÜ_t(‚Üíx_0); ‚Üíőł\big) - \big(‚Üíx_1-(1-ŌÉ_\textrm{min})‚Üíx_0\big)\big\|^2.$$

# Conditional Flow Matching Algorithm

![w=88%,mw=19%,h=right,f=right](imagen_architecture_overall.svgz)

Let $v_t(‚Üíx; ‚Üíőł)$ be a U-Net-like neural network model predicting the vector
field.

~~~
### Training

During training, we minimize CFM objective by SGD/Adam:
$$ūĚďõ_\textrm{CFM}(‚Üíőł) ‚ČĚ ūĚĒľ_{t, ‚Üíx_1 ‚ąľ p_\textrm{data}, ‚Üíx_0 ‚ąľ p_0} \big\|v_t\big(ŌÜ_t(‚Üíx_0); ‚Üíőł\big) - \big(‚Üíx_1-(1-ŌÉ_\textrm{min})‚Üíx_0\big)\big\|^2.$$

~~~
Specifically, we generate batches of training data $‚áČX ‚ąľ p_\textrm{data}$ as
usual, for each batch example we also generate $t ‚ąľ U[0, 1]$ and $‚Üíx_0 ‚ąľ ūĚďĚ(‚Üí0,
‚áČI)$, and we minimize the mean squared error
$$\|v_t\big(ŌÜ_t(‚Üíx_0); ‚Üíőł\big) - \big(‚Üíx_1-(1-ŌÉ_\textrm{min})‚Üíx_0\big)\big\|^2.$$

~~~
### Sampling

In order to generate an image, we start by sampling $‚Āá‚Üíx_0 ‚ąľ p_0$, and then
perform numerical integration by the Euler method using $T$ steps by
$$‚Üíx_{t+\frac{1}{T}} ‚Üź ‚Üíx_t + \tfrac{1}{T} v_t(‚Üíx_t; ‚Üíőł).$$

# Conditional Models, Classifier-Free Guidance

In many cases we want the generative model to be conditional.
~~~
We have already seen how to condition it on the current time step. Additionally,
we might consider also conditioning on
- an image (e.g., for super-resolution): the image is then resized and
  concatenated with the input noised image (and optionally in other places,
  like after every resolution change);

~~~
- a text: the usual approach is to encode the text using some pre-trained
  encoder, and then to introduce an ‚Äúimage-text‚ÄĚ attention layer (usually
  after the self-attention layers).

~~~
To make the effect of conditioning stronger during sampling, we might
also employ _classifier-free guidance_:
~~~
- During training, we sometimes train $v_t(‚Üíx, y; ‚Üíőł)$ with the conditioning $y$,
  and sometimes we train $v_t(‚Üíx, \varnothing; ‚Üíőł)$ without the conditioning.
~~~
- During sampling, we pronounce the effect of the conditioning by taking
  the unconditioned vector field and adding the difference between conditioned and
  unconditioned vector field _weighted by the weight_ $w$ (values like $w=5$ or
  $w=7.5$ are mentioned in papers):
  $$v_t(‚Üíx, \varnothing; ‚Üíőł) + w\big(v_t(‚Üíx, y; ‚Üíőł) - v_t(‚Üíx_t, \varnothing; ‚Üíőł)\big).$$

# DDPM ‚Äď Deriving Loss using Jensen's Inequality

$\displaystyle -ūĚĒľ_{q(‚Āá‚Üíx_0)}\big[\log p_{‚Üíőł}(‚Āá‚Üíx_0)\big] = -ūĚĒľ_{q(‚Āá‚Üíx_0)}\big[\log ūĚĒľ_{p_{‚Üíőł}(‚Āá‚Üíx_{1:T})}[p_{‚Üíőł}(‚Āá‚Üíx_0|‚Üíx_{1:T})]\big]$

~~~
$\displaystyle \kern2em{} = -ūĚĒľ_{q(‚Āá‚Üíx_0)}\Big[\log ūĚĒľ_{q(‚Āá‚Üíx_{1:T}|‚Āá‚Üíx_0)}\Big[\tfrac{p_{‚Üíőł}(‚Āá‚Üíx_{0:T})}{q(‚Āá‚Üíx_{1:T}|‚Āá‚Üíx_0)}\Big]\Big]$

~~~
$\displaystyle \kern2em{} ‚Č§ -ūĚĒľ_{q(‚Āá‚Üíx_{0:T})}\Big[\log \tfrac{p_{‚Üíőł}(‚Āá‚Üíx_{0:T})}{q(‚Āá‚Üíx_{1:T}|‚Āá‚Üíx_0)}\Big] = ūĚĒľ_{q(‚Āá‚Üíx_{0:T})}\Big[\log \tfrac{q(‚Āá‚Üíx_{1:T}|‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_{0:T})}\Big]$

~~~
$\displaystyle \kern2em{} = ūĚĒľ_{q(‚Āá‚Üíx_{0:T})}\Big[-\log p_{‚Üíőł}(‚Āá‚Üíx_T) + ‚ąĎ\nolimits_{t=2}^\T\log \tfrac{q(‚Āá‚Üíx_t|‚Āá‚Üíx_{t-1})}{p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)} + \log \tfrac{q(‚Āá‚Üíx_1|‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_0|‚Āá‚Üíx_1)}\Big]$

~~~
$\displaystyle \kern2em{} = ūĚĒľ_{q(‚Āá‚Üíx_{0:T})}\Big[-\log p_{‚Üíőł}(‚Āá‚Üíx_T) + ‚ąĎ\nolimits_{t=2}^\T\log \Big(\tfrac{q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t,‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)}\tfrac{q(‚Āá‚Üíx_t|‚Āá‚Üíx_0)}{q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_0)}\Big) + \log \tfrac{q(‚Āá‚Üíx_1|‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_0|‚Āá‚Üíx_1)}\Big]$

~~~
$\displaystyle \kern2em{} = ūĚĒľ_{q(‚Āá‚Üíx_{0:T})}\Big[-\log p_{‚Üíőł}(‚Āá‚Üíx_T) + ‚ąĎ\nolimits_{t=2}^\T\log \tfrac{q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t,‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)} + \log \tfrac{q(‚Āá‚Üíx_T|‚Āá‚Üíx_0)}{q(‚Āá‚Üíx_1|‚Āá‚Üíx_0)} + \log \tfrac{q(‚Āá‚Üíx_1|‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_0|‚Āá‚Üíx_1)}\Big]$

~~~
$\displaystyle \kern2em{} = ūĚĒľ_{q(‚Āá‚Üíx_{0:T})}\Big[\log \tfrac{q(‚Āá‚Üíx_T|‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_T)} + ‚ąĎ\nolimits_{t=2}^\T\log \tfrac{q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t,‚Āá‚Üíx_0)}{p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)} -\log p_{‚Üíőł}(‚Āá‚Üíx_0|‚Āá‚Üíx_1)\Big]$

~~~
$\displaystyle \kern2em{} = ūĚĒľ_{q(‚Āá‚Üíx_{0:T})}\Big[\underbrace{\scriptsize D_\textrm{KL}(q(‚Āá‚Üíx_T|‚Āá‚Üíx_0) \| p_{‚Üíőł}(‚Āá‚Üíx_T))}_{L_T} + ‚ąĎ\nolimits_{t=2}^\T\underbrace{\scriptsize D_\textrm{KL}(q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t,‚Āá‚Üíx_0) \| p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)}_{L_t} \underbrace{-\log p_{‚Üíőł}(‚Āá‚Üíx_0|‚Āá‚Üíx_1)}_{L_0}\Big]$

---
# DDPM ‚Äď Deriving Loss using Jensen's Inequality

The whole loss is therefore composed of the following components:

~~~
- $L_T = D_\textrm{KL}\big(q(‚Āá‚Üíx_T|‚Āá‚Üíx_0) \| p_{‚Üíőł}(‚Āá‚Üíx_T)\big)$ is constant
  with respect to $‚Üíőł$ and can be ignored,

~~~
- $L_t = D_\textrm{KL}\big(q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t,‚Āá‚Üíx_0) \| p_{‚Üíőł}(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t)\big)$
  is KL divergence between two Gaussians, so it can be computed explicitly as

  $$L_t = ūĚĒľ\bigg[\frac{1}{2\|ŌÉ_t‚áČI\|^2}\Big\| ‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0) - ‚Üíőľ_{‚Üíőł}(‚Āá‚Üíx_t, t) \Big\|^2\bigg],$$
~~~
- $L_0 = -\log p_{‚Üíőł}(‚Āá‚Üíx_0|‚Āá‚Üíx_1)$ can be used to generate discrete $‚Āá‚Üíx_0$
  from the continuous $‚Āá‚Üíx_1$; we will ignore it in the slides for simplicity.

---
# DDPM ‚Äď Reparametrizing Model Prediction

Recall that $q(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t, ‚Āá‚Üíx_0) = ūĚďĚ\big(‚Āá‚Üíx_{t-1}; \textcolor{blue}{‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0)}, \textcolor{green}{ő≤ŐÉ_t}‚áČI\big)$ for

$\displaystyle\kern5em{}\mathllap{\textcolor{blue}{‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0)}} = \frac{\sqrt{őĪŐĄ_{t-1}}ő≤_t}{1-őĪŐĄ_t}‚Āá‚Üíx_0 + \frac{\sqrt{őĪ_t}(1-őĪŐĄ_{t-1})}{1-őĪŐĄ_t}‚Āá‚Üíx_t,$

$\displaystyle\kern5em{}\mathllap{\textcolor{green}{ő≤ŐÉ_t}} = \frac{1-őĪŐĄ_{t-1}}{1-őĪŐĄ_t}ő≤_t.$

~~~
Because $‚Āá‚Üíx_t = \sqrt{őĪŐĄ_t} ‚Āá‚Üíx_0 + \sqrt{1-őĪŐĄ_t}‚Āá‚Üíe_t$, we get $‚Āá‚Üíx_0 = \textcolor{red}{\frac{1}{\sqrt{őĪŐĄ_t}}\big(‚Āá‚Üíx_t - \sqrt{1-őĪŐĄ_t}‚Āá‚Üíe_t\big)}$.

~~~
Substituting $‚Āá‚Üíx_0$ to $‚ÜíőľŐÉ_t$, we get

$\displaystyle\kern5em{}\mathllap{\textcolor{blue}{‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0)}} = \frac{\sqrt{ŠĺĪ_{t-1}}ő≤_t}{1-ŠĺĪ_t}\textcolor{red}{\frac{1}{\sqrt{ŠĺĪ_t}}\Big(‚Āá‚Üíx_t - \sqrt{1-ŠĺĪ_t}‚Āá‚Üíe_t\Big)} + \frac{\sqrt{őĪ_t}(1-ŠĺĪ_{t-1})}{1-ŠĺĪ_t}‚Āá‚Üíx_t$

~~~
$\displaystyle\kern5em{} = \Big(\frac{\sqrt{ŠĺĪ_{t-1}}ő≤_t}{1-ŠĺĪ_t}\frac{1}{\sqrt{ŠĺĪ_t}} + \frac{\sqrt{őĪ_t}(1-ŠĺĪ_{t-1})}{1-ŠĺĪ_t}\Big)‚Āá‚Üíx_t - \Big(\frac{\sqrt{ŠĺĪ_{t-1}}ő≤_t}{1-ŠĺĪ_t}\frac{\sqrt{1-ŠĺĪ_t}}{\sqrt{ŠĺĪ_t}}\Big)‚Āá‚Üíe_t$

~~~
$\displaystyle\kern5em{} = \frac{ő≤_t + őĪ_t(1-őĪŐĄ_{t-1})}{(1-ŠĺĪ_t)\sqrt{őĪ_t}}‚Āá‚Üíx_t - \Big(\frac{ő≤_t}{\sqrt{1-ŠĺĪ_t}\sqrt{őĪ_t}}\Big)‚Āá‚Üíe_t = \textcolor{blue}{\frac{1}{\sqrt{őĪ_t}}\Big(‚Āá‚Üíx_t - \frac{1-őĪ_t}{\sqrt{1-őĪŐĄ_t}}‚Āá‚Üíe_t\Big)}.$

# DDPM ‚Äď Reparametrizing Model Prediction

We change our model to predict $‚ÜíőĶ_{‚Üíőł}(‚Āá‚Üíx_t, t)$ instead of
$‚Üíőľ_{‚Üíőł}(‚Āá‚Üíx_t, t)$.
~~~
The loss $L_t$ then becomes

$\displaystyle\kern3em{}\mathllap{L_t} = ūĚĒľ\bigg[\frac{1}{2\|ŌÉ_t‚áČI\|^2}\Big\| \textcolor{blue}{‚ÜíőľŐÉ_t(‚Āá‚Üíx_t, ‚Āá‚Üíx_0)} - \textcolor{green}{‚Üíőľ_{‚Üíőł}(‚Āá‚Üíx_t, t)} \Big\|^2\bigg]$

~~~
$\displaystyle\kern3em{} = ūĚĒľ\bigg[\frac{1}{2\|ŌÉ_t‚áČI\|^2}\Big\| \textcolor{blue}{\frac{1}{\sqrt{őĪ_t}}\Big(‚Āá‚Üíx_t - \frac{1-őĪ_t}{\sqrt{1-őĪŐĄ_t}}‚Āá‚Üíe_t\Big)} - \textcolor{green}{\frac{1}{\sqrt{őĪ_t}}\Big(‚Āá‚Üíx_t - \frac{1-őĪ_t}{\sqrt{1-őĪŐĄ_t}}‚ÜíőĶ_{‚Üíőł}(‚Āá‚Üíx_t, t)\Big)} \Big\|^2\bigg]$

~~~
$\displaystyle\kern3em{} = ūĚĒľ\bigg[\frac{(1-őĪ_t)^2}{2őĪ_t(1-őĪŐĄ_t)\|ŌÉ_t‚áČI\|^2}\Big\| ‚Āá‚Üíe_t - ‚ÜíőĶ_{‚Üíőł}(‚Āá‚Üíx_t, t) \Big\|^2\bigg]$

~~~
$\displaystyle\kern3em{} = ūĚĒľ\bigg[\frac{(1-őĪ_t)^2}{2őĪ_t(1-őĪŐĄ_t)\|ŌÉ_t‚áČI\|^2}\Big\| ‚Āá‚Üíe_t - ‚ÜíőĶ_{‚Üíőł}\big(\sqrt{ŠĺĪ_t} ‚Āá‚Üíx_0 + \sqrt{1-ŠĺĪ_t}‚Āá‚Üíe_t, t\big) \Big\|^2\bigg].$

~~~
The authors found that training without the weighting term performs better, so

$$L_t^\textrm{simple} = ūĚĒľ_{t‚ąą\{1..T\},‚Āá‚Üíx_0,‚Āá‚Üíe_t}\Big[\big\| ‚Āá‚Üíe_t - ‚ÜíőĶ_{‚Üíőł}\big(\sqrt{ŠĺĪ_t} ‚Āá‚Üíx_0 + \sqrt{1-ŠĺĪ_t}‚Āá‚Üíe_t, t\big) \big\|^2\Big].$$

~~~
Note that both losses have the same optimum if we used independent $‚ÜíőĶ_{‚Üíőł_t}$ for every $t$.

# Denoising Diffusion Implicit Models

We now describe _Denoising Diffusion Implicit Models (DDIM)_, which utilize
a different forward process.

~~~
This forward process is designed to:
- allow faster sampling,

~~~
- have the same ‚Äúmarginals‚ÄĚ $q(‚Āá‚Üíx_t | ‚Āá‚Üíx_0) = ūĚďĚ\big(\sqrt{őĪŐĄ_t}‚Āá‚Üíx_0, (1-őĪŐĄ_t)‚áČI\big)$.

~~~
The second condition will allow us to use the same loss as in DDPM; therefore,
the training algorithm is exactly identical do DDPM, only the sampling algorithm
is different.

~~~
Note that in the slides, only a special case of DDIM is described; the original
paper describes a more general forward process. However, the special case
presented here is almost exclusively used.

---
# Denoising Diffusion Implicit Models ‚Äď The Forward Process

The forward process of DDIM can be described using

$$q_0(‚Āá‚Üíx_{1:T}|‚Āá‚Üíx_0) = q_0(‚Āá‚Üíx_T | ‚Āá‚Üíx_0) ‚ąŹ\nolimits_{t=2}^\T q_0(‚Āá‚Üíx_{t-1}|‚Āá‚Üíx_t, ‚Āá‚Üíx_0),$$

where

~~~
- $q_0(‚Āá‚Üíx_T | ‚Āá‚Üíx_0) = ūĚďĚ\big(\sqrt{őĪŐĄ_T}‚Āá‚Üíx_0, (1-őĪŐĄ_T)‚áČI\big)$,

~~~
- $q_0(‚Āá‚Üíx_{t-1} | ‚Āá‚Üíx_t, ‚Āá‚Üíx_0) = ūĚďĚ\Big(\sqrt{őĪŐĄ_{t-1}}‚Āá‚Üíx_0 + \sqrt{1-őĪŐĄ_{t-1}}\big(\textcolor{green}{\tfrac{‚Āá‚Üíx_t - \sqrt{őĪŐĄ_t}‚Āá‚Üíx_0}{\sqrt{1-őĪŐĄ_t}}}\big), 0‚čÖ‚áČI\Big)$.

~~~
With these definitions, we can prove by induction that $q_0(‚Āá‚Üíx_t | ‚Āá‚Üíx_0) = ūĚďĚ\big(\sqrt{őĪŐĄ_t}‚Āá‚Üíx_0, (1-őĪŐĄ_t)‚áČI\big)$:

~~~
$\displaystyle\kern3em{}\mathllap{‚Āá‚Üíx_{t-1}} = \sqrt{ŠĺĪ_{t-1}}‚Āá‚Üíx_0 + \sqrt{1-ŠĺĪ_{t-1}}\big(\tfrac{\textcolor{blue}{‚Āá‚Üíx_t} - \sqrt{ŠĺĪ_t}‚Āá‚Üíx_0}{\sqrt{1-ŠĺĪ_t}}\big)$

~~~
$\displaystyle\kern3em{} = \sqrt{ŠĺĪ_{t-1}}‚Āá‚Üíx_0 + \sqrt{1-ŠĺĪ_{t-1}}\big(\tfrac{\textcolor{blue}{\sqrt{őĪŐĄ_t}‚Āá‚Üíx_0 + \sqrt{1-őĪŐĄ_t}‚Āá‚Üíe_t} - \sqrt{ŠĺĪ_t}‚Āá‚Üíx_0}{\sqrt{1-ŠĺĪ_t}}\big) = \sqrt{ŠĺĪ_{t-1}}‚Āá‚Üíx_0 + \sqrt{1-ŠĺĪ_{t-1}}\textcolor{green}{‚Āá‚Üíe_t}$.

~~~
The real ‚Äúforward‚ÄĚ $q_0(‚Āá‚Üíx_t | ‚Āá‚Üíx_{t-1}, ‚Āá‚Üíx_0)$ can be expressed using Bayes'
theorem using the above definition, but we do not actually need it.

# Denoising Diffusion Implicit Models ‚Äď The Reverse Process

The definition of $q_0(‚Āá‚Üíx_{t-1} | ‚Āá‚Üíx_t, ‚Āá‚Üíx_0)$ provides us also with
a sampling algorithm; after sampling the initial noise $‚Āá‚Üíx_T ‚ąľ ūĚďĚ(‚Üí0, ‚áČI)$,
we perform the following for $t$ from $T$ down to 1:
$$\begin{aligned}
‚Üíx_{t-1} 
  &= \sqrt{őĪŐĄ_{t-1}}\textcolor{blue}{‚Āá‚Üíx_0} + \sqrt{1-őĪŐĄ_{t-1}}‚ÜíőĶ_{‚Üíőł}(‚Üíx_t, t) \\
  &= \sqrt{őĪŐĄ_{t-1}}\Big(\textcolor{blue}{\tfrac{‚Üíx_t-\sqrt{1-őĪŐĄ_t}‚ÜíőĶ_{‚Üíőł}(‚Üíx_t, t)}{\sqrt{őĪŐĄ_t}}}\Big) + \sqrt{1-őĪŐĄ_{t-1}}‚ÜíőĶ_{‚Üíőł}(‚Üíx_t, t).
\end{aligned}$$

~~~
An important property of $q_0$ is that it can also model several
steps at once:

![w=55%,h=center](ddim_accelerated_generation.png)

$$q_0(‚Āá‚Üíx_{t'} | ‚Āá‚Üíx_t, ‚Āá‚Üíx_0) = ūĚďĚ\big(\sqrt{őĪŐĄ_{t'}}‚Āá‚Üíx_0 + \sqrt{1-őĪŐĄ_{t'}}\big(\tfrac{‚Āá‚Üíx_t - \sqrt{ŠĺĪ_t}‚Āá‚Üíx_0}{\sqrt{1-ŠĺĪ_t}}\big), ‚áČ0\big).$$

---
# Denoising Diffusion Implicit Models ‚Äď Accelerated Sampling

We base our accelerated sampling algorithm on the ‚Äúmultistep‚ÄĚ $q_0(‚Āá‚Üíx_{t'} | ‚Āá‚Üíx_t, ‚Āá‚Üíx_0)$.

~~~
Let $t_S=T, t_{S-1}, ‚Ä¶, t_1$ be a subsequence of the process steps (usually,
a uniform subsequence of $T, ‚Ä¶, 1$ is used), and let $t_0=0$.
~~~
Starting from initial noise $‚Āá‚Üíx_T ‚ąľ ūĚďĚ(‚Üí0, ‚áČI)$, we perform $S$ sampling
steps for $i$ from $S$ down to 1:

$$‚Üíx_{t_{i-1}} ‚Üź \sqrt{őĪŐĄ_{t_{i-1}}}\Big(\underbrace{\tfrac{‚Üíx_{t_i}-\sqrt{1-őĪŐĄ_{t_i}}‚ÜíőĶ_{‚Üíőł}(‚Üíx_{t_i}, t_i)}{\sqrt{őĪŐĄ_{t_i}}}}_{‚Üíx_0\textrm{~estimate}}\Big) + \sqrt{1-őĪŐĄ_{t_{i-1}}}‚ÜíőĶ_{‚Üíőł}(‚Üíx_{t_i}, t_i).$$

~~~
The sampling procedure can be described in words as follows:
- using the current time step $t_i$, we compute the estimated
  noise $‚ÜíőĶ_{‚Üíőł}(‚Üíx_{t_i}, t_i)$;
~~~
- by utilizing the current signal rate $\sqrt{őĪŐĄ_{t_i}}$ and noise
  rate $\sqrt{1-őĪŐĄ_{t_i}}$, we estimate $‚Āá‚Üíx_0$;
~~~
- we obtain $‚Üíx_{t_{i-1}}$ by combining the estimated signal $‚Āá‚Üíx_0$ and
  noise $‚ÜíőĶ_{‚Üíőł}(‚Üíx_{t_i}, t_i)$ using the signal and noise rates of the time
  step $t_{i-1}$.

---
style: .katex-display { margin: .4em 0 }
# Denoising Diffusion Implicit Models ‚Äď Accelerated Sampling

For comparison, we show both the original $\textcolor{#2c2}{\textrm{DDPM}}$ and the new
$\textcolor{#d4d}{\textrm{DDIM}}$ sampling algorithms:

~~~
- sample $‚Üíx_T$ from $ūĚďĚ(‚Üí0, ‚ÜíI)$

~~~
- let $t_S=T, t_{S-1}, ‚Ä¶, t_1=1$ be a subsequence of the process steps
  - $\textcolor{#2c2}{\textrm{DDPM}}$: the original sequence $\textcolor{#2c2}{T, ‚Ä¶, 1}$ is usually used
  - $\textcolor{#d4d}{\textrm{DDIM}}$: $S$ regularly-spaced steps $\textcolor{#d4d}{T, \frac{S-1}{S}T, \frac{S-2}{S}T, ‚Ä¶, 1}$ are usually used
  - additionally, we define $t_0 = 0$

~~~
- for $i=S, ‚Ä¶, 1$:
  $$\begin{aligned}
  \textcolor{#2c2}{\textrm{DDPM}}:\kern2em &
    ‚Üíx_{t_{i-1}} ‚Üź \textcolor{#2c2}{\sqrt{\tfrac{1}{őĪ_{t_i}}}} \bigg(‚Üíx_{t_i} - \textcolor{#2c2}{\tfrac{1-őĪ_{t_i}}{\sqrt{1-őĪŐĄ_{t_i}}}}‚ÜíőĶ_{‚Üíőł}(‚Üíx_{t_i}, t_i) \bigg) \textcolor{#2c2}{+ ŌÉ_t ‚Üíz_t}\\
  \textcolor{#d4d}{\textrm{DDIM}}:\kern2em &
    ‚Üíx_{t_{i-1}} ‚Üź \textcolor{#d4d}{\sqrt{őĪŐĄ_{t_{i-1}}}} \bigg(\underbrace{\tfrac{‚Üíx_{t_i} - \textcolor{#d4d}{\sqrt{1-őĪŐĄ_{t_i}}}‚ÜíőĶ_{‚Üíőł}(‚Üíx_{t_i}, t_i)}{\textcolor{#d4d}{\sqrt{őĪŐĄ_{t_i}}}}}_{‚Üíx_0\textrm{~estimate}} \bigg) + \textcolor{#d4d}{\sqrt{1-őĪŐĄ_{t_{i-1}}}‚ÜíőĶ_{‚Üíőł}(‚Üíx_{t_i}, t_i)} \\
  \end{aligned}$$

~~~
- return $‚Üíx_0$

# Score Matching

Recall that loglikelihood-based models explicit represent the density
function, commonly using an unnormalized probabilistic model
$$p_{‚Üíőł}(‚Āá‚Üíx) = \frac{e^{f_{‚Üíőł}(‚Āá‚Üíx)}}{Z_{‚Üíőł}},$$
and it is troublesome to ensure the tractability of the normalization constant
$Z_{‚Üíőł}$.

~~~
One way how to avoid the normalization is to avoid the explicit density
$p_{‚Üíőł}(‚Āáx)$, and represent a **score function** instead, where the score
function is the gradient of the log density:
$$s_{‚Üíőł}(‚Āá‚Üíx) = ‚ąá_{‚Āá‚Üíx} \log p_{‚Üíőł}(‚Āá‚Üíx),$$

~~~
because
$$s_{‚Üíőł}(‚Āá‚Üíx) = ‚ąá_{‚Āá‚Üíx} \log p_{‚Üíőł}(‚Āá‚Üíx) = ‚ąá_{‚Āá‚Üíx} \log \frac{e^{f_{‚Üíőł}(‚Āá‚Üíx)}}{Z_{‚Üíőł}} = ‚ąá_{‚Āá‚Üíx} f_{‚Üíőł}(‚Āá‚Üíx) - \underbrace{‚ąá_{‚Āá‚Üíx} \log Z_{‚Üíőł}}_0 = ‚ąá_{‚Āá‚Üíx} f_{‚Üíőł}(‚Āá‚Üíx).$$

---
# Langevin Dynamics

When we have a score function $‚ąá_{‚Āá‚Üíx} \log p_{‚Üíőł}(‚Āá‚Üíx)$, we can use it to
perform sampling from the distribution $p_{‚Üíőł}(‚Āá‚Üíx)$ by using **Langevin
dynamics**, which is an algorithm akin to SGD, but performing sampling
instead of optimum finding.
~~~
Starting with $‚Āá‚Üíx_0$, we iteratively set
$$‚Āá‚Üíx_{i+1} ‚Üź ‚Āá‚Üíx_i + őĶ‚ąá_{‚Āá‚Üíx_i} \log p_{‚Üíőł}(‚Āá‚Üíx_i) + \sqrt{2őĶ}\,‚Āá‚Üíz_i,\textrm{~~where~~}‚Üíz_i ‚ąľ ūĚďĚ(‚Üí0, ‚áČI).$$

~~~
![w=31%,f=right](langevin_dynamics.gif)

When $őĶ ‚Üí 0$ and $K ‚Üí ‚ąě$, $‚Āá‚Üíx_K$ obtained by the Langevin dynamics
converges to a sample from the distribution $p_{‚Üíőł}(‚Āá‚Üíx)$.

# Noise Conditional Score Network

However, estimating the score function from data is inaccurate
in low-density regions.

![w=70%,mw=90%,h=center](score_estimation_noiseless.png)

~~~
In order to accurately estimate the score function in low-density
regions, we perturb the data distribution by isotropic Gaussian noise
with various noise rates $ŌÉ_t$:
$$q_{ŌÉ_t}(‚Āá‚ÜíxŐÉ) ‚ČĚ ūĚĒľ_{‚Āá‚Üíx ‚ąľ p(‚Āá‚Üíx)} \big[ūĚďĚ(‚Āá‚ÜíxŐÉ; ‚Āá‚Üíx, {ŌÉ_t}^2 ‚áČI)\big],$$
~~~
where the noise distribution $q_{ŌÉ_t}(‚Āá‚ÜíxŐÉ|‚Āá‚Üíx) = ūĚďĚ(‚Āá‚ÜíxŐÉ;‚Āá‚Üíx, ŌÉ_t^2 ‚áČI)$
as analogous to the forward process in the diffusion models.

---
style: .katex-display { margin: .8em 0 }
# Noise Conditional Score Network

To train the score function $‚Üís_{‚Üíőł}(‚Āá‚Üíx, ŌÉ_t) = ‚ąá_{‚Āá‚Üíx} \log q_{ŌÉ_t}(‚Āá‚Üíx)$, we need to minimize
the following objective:
$$ūĚĒľ_{t, ‚Āá‚ÜíxŐÉ ‚ąľ q_{ŌÉ_t}}\Big[\big\|‚Üís_{‚Üíőł}(‚Āá‚ÜíxŐÉ, ŌÉ_t) - ‚ąá_{‚Āá‚ÜíxŐÉ} \log q_{ŌÉ_t}(‚Āá‚ÜíxŐÉ)\big\|^2\Big].$$

~~~
It can be shown (see _P. Vincent: A connection between score matching and
denoising autoencoders_) that it is equivalent to minimize the _denoising score
matching_ objective:
$$ūĚĒľ_{t, ‚Āá‚Üíx ‚ąľ p(‚Āá‚Üíx), ‚Āá‚ÜíxŐÉ ‚ąľ q_{ŌÉ_t}(‚Āá‚ÜíxŐÉ|‚Āá‚Üíx)}\Big[\big\|‚Üís_{‚Üíőł}(‚Āá‚ÜíxŐÉ, ŌÉ_t) - ‚ąá_{‚ÜíxŐÉ} \log q_{ŌÉ_t}(‚Āá‚ÜíxŐÉ|‚Āá‚Üíx)\big\|^2\Big].$$

~~~
In our case, $‚ąá_{‚Āá‚ÜíxŐÉ} \log q_{ŌÉ_t}(‚Āá‚ÜíxŐÉ|‚Āá‚Üíx) = ‚ąá_{‚Āá‚ÜíxŐÉ} \frac{-\|‚Āá‚ÜíxŐÉ-‚Āá‚Üíx\|^2}{2ŌÉ_t^2} = -\frac{‚Āá‚ÜíxŐÉ-‚Āá‚Üíx}{ŌÉ_t^2}$.
~~~
Because $‚Āá‚ÜíxŐÉ = ‚Āá‚Üíx + ŌÉ_t ‚Āá‚Üíe$ for standard normal random variable $‚Āá‚Üíe ‚ąľ ūĚďĚ(‚Üí0,
‚áČI)$, we can rewrite the objective to

$$ūĚĒľ_{t, ‚Āá‚Üíx ‚ąľ p(‚Āá‚Üíx), ‚Āá‚Üíe ‚ąľ ūĚďĚ(‚Üí0, ‚áČI)}\Big[\big\|‚Üís_{‚Üíőł}(‚Āá‚Üíx+ŌÉ_t ‚Āá‚Üíe, ŌÉ_t) - \frac{-‚Āá‚Üíe}{ŌÉ_t}\big\|^2\Big],$$

so the score function basically estimates the noise given a noised image.

---

Once we have trained the score function for various noise rates $ŌÉ_t$, we can
sample using annealed Langevin dynamics, where we utilize using gradually
