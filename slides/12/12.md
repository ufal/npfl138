# Generative Models

Generative models are given a set of realizations of a random variable $вҒҮвҶ’x$ and
their goal is to estimate $P(вҶ’x)$.

~~~
Usually the goal is to be able to sample from $P(вҒҮвҶ’x)$, but sometimes an
explicit calculation of $P(вҶ’x)$ is also possible.

---
# Deep Generative Models

![w=25%,h=center](generative_model.svgz)

One possible approach to estimate $P(вҶ’x)$ is to assume that the random variable
$вҒҮвҶ’x$ depends on a **latent variable** $вҒҮвҶ’z$:
$$P(вҶ’x) = вҲ‘_{вҶ’z} P(вҶ’z) P(вҶ’x | вҶ’z) = рқ”ј_{вҶ’z вҲј P(вҒҮвҶ’z)} P(вҶ’x | вҶ’z).$$

~~~
We use neural networks to estimate the conditional probability
$P_{вҶ’Оё}(вҶ’x | вҶ’z)$.

# AutoEncoders

![w=50%,h=center](ae.svgz)

- Autoencoders are useful for unsupervised feature extraction, especially when
  performing input compression (i.e., when the dimensionality of the latent
  space $вҶ’z$ is smaller than the dimensionality of the input).

~~~
- When $вҶ’x + вҶ’Оө$ is used as input, autoencoders can perform denoising.

~~~
- However, the latent space $вҶ’z$ does not need to be fully covered, so
  a randomly chosen $вҶ’z$ does not need to produce a valid $вҶ’x$.

# Variational AutoEncoders

We assume $P(вҒҮвҶ’z)$ is fixed and independent on $вҒҮвҶ’x$.

~~~
We approximate $P(вҶ’x | вҶ’z)$ using a neural network $P_{вҶ’Оё}(вҶ’x | вҶ’z)$, the
**decoder**.

~~~
However, in order to train an autoencoder, we need to know the
вҖңinverseвҖқ $P_{вҶ’Оё}(вҶ’z | вҶ’x)$, which cannot be usually computed directly.

~~~
Therefore, we approximate $P_{вҶ’Оё}(вҶ’z | вҶ’x)$ by another trainable neural network
$Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)$, the **encoder**.

# Jensen's Inequality

To derive a loss for training variational autoencoders, we first formulate
the Jensen's inequality.

~~~
![w=95%,mw=37%,h=right,f=right](../02/convex_2d.svgz)

Recall that convex functions by definition fulfil that for $вҶ’u, вҶ’v$ and real $0
вүӨ t вүӨ 1$,
$$f(tвҶ’u + (1-t)вҶ’v) вүӨ tf(вҶ’u) + (1-t)f(вҶ’v).$$

~~~
The **Jensen's inequality** generalizes the above property to any _convex_
combination of points: if we have $вҶ’u_i вҲҲ в„қ^D$ and weights $w_i вҲҲ в„қ^+$ such
that $вҲ‘_i w_i = 1$, it holds that

![w=95%,mw=37%,h=right,f=right](jensens_inequality.png)

$$f\big(вҲ‘_i w_i вҶ’u_i\big) вүӨ вҲ‘_i w_i f\big(вҶ’u_i\big).$$

~~~
The Jensen's inequality can be formulated also for probability distributions
(whose expectation can be considered an infinite convex combination):

$$f\big(рқ”ј[вҒҮвҶ’u]\big) вүӨ рқ”ј_{вҒҮвҶ’u} \big[f(вҒҮвҶ’u)\big].$$

# VAE вҖ“ Loss Function Derivation

Our goal will be to maximize the log-likelihood as usual, but we need to express
it using the latent variable $вҶ’z$:
$$\log P_{вҶ’Оё}(вҶ’x) = \log рқ”ј_{P(вҶ’z)} \big[P_{вҶ’Оё}(вҶ’x | вҶ’z)\big].$$

~~~
However, approximating the expectation using a single sample has monstrous
variance, because for most $вҶ’z$, $P_{вҶ’Оё}(вҶ’x | вҶ’z)$ will be nearly zero.

~~~
We therefore turn to our _encoder_, which is able for a given $вҶ’x$ to generate
вҖңitsвҖқ $вҶ’z$:

~~~
$\displaystyle \kern6em\mathllap{\log P_{вҶ’Оё}(вҶ’x)} = \log рқ”ј_{P(вҶ’z)} \big[P_{вҶ’Оё}(вҶ’x | вҶ’z)\big]$

~~~
$\displaystyle \kern6em{} = \log рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z|вҶ’x)} \bigg[P_{вҶ’Оё}(вҶ’x | вҶ’z) вӢ… \frac{P(вҶ’z)}{Q_{вҶ’ПҶ}(вҶ’z|вҶ’x)}\bigg]\kern2.7em\textcolor{gray}{\textrm{\footnotesize now we use the Jensen's inequality}}$

~~~
$\displaystyle \kern6em{} вүҘ рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z|вҶ’x)} \bigg[\log P_{вҶ’Оё}(вҶ’x | вҶ’z) + \log\frac{P(вҶ’z)}{Q_{вҶ’ПҶ}(вҶ’z|вҶ’x)}\bigg]$

~~~
$\displaystyle \kern6em{} = рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z|вҶ’x)} \big[\log P_{вҶ’Оё}(вҶ’x | вҶ’z)\big] - D_\textrm{KL}\big(Q_{вҶ’ПҶ}(вҶ’z|вҶ’x) \| P(вҶ’z)\big) вүқ рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x).$

---
# VAE вҖ“ Variational (or Evidence) Lower Bound

The resulting **variational lower bound** or **evidence lower bound** (ELBO),
denoted $рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x)$, can be also defined explicitly as:
$$рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x) = \log P_{вҶ’Оё}(вҶ’x) - D_\textrm{KL}\big(Q_{вҶ’ПҶ}(вҶ’z | вҶ’x) \| P_{вҶ’Оё}(вҶ’z | вҶ’x)\big).$$

~~~
Because KL-divergence is nonnegative, $рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x) вүӨ \log P_{вҶ’Оё}(вҶ’x).$

~~~
By using simple properties of conditional and joint probability, we get that

~~~
$\displaystyle \kern9em\mathllap{рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x)} = рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)} \big[\log P_{вҶ’Оё}(вҶ’x) + \log P_{вҶ’Оё}(вҶ’z | вҶ’x) - \log Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)\big]$

~~~
$\displaystyle \kern9em{} = рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)} \big[\log P_{вҶ’Оё}(вҶ’x, вҶ’z) - \log Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)\big]$

~~~
$\displaystyle \kern9em{} = рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)} \big[\log P_{вҶ’Оё}(вҶ’x | вҶ’z) + \log P(вҶ’z) - \log Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)\big]$

~~~
$\displaystyle \kern9em{} = рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)} \big[\log P_{вҶ’Оё}(вҶ’x | вҶ’z)\big] - D_\textrm{KL}\big(Q_{вҶ’ПҶ}(вҶ’z | вҶ’x) \| P(вҶ’z)\big).$

# Variational AutoEncoders Training

$$-рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x) = рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)} \big[-\log P_{вҶ’Оё}(вҶ’x | вҶ’z)\big] + D_\textrm{KL}\big(Q_{вҶ’ПҶ}(вҶ’z | вҶ’x) \| P(вҶ’z)\big)$$

- We train a VAE by minimizing the $-рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x)$.

~~~
- The $рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)}$ is estimated using a single sample.
~~~
- The distribution $Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)$ is parametrized as a normal distribution
  $рқ“қ(вҶ’z | вҶ’Ој, вҶ’Пғ^2)$, with the model predicting $вҶ’Ој$ and $вҶ’Пғ$ given $вҶ’x$.
~~~
  - In order for $вҶ’Пғ$ to be positive, we can use $\exp$ activation function
    (so that the network predicts $\log вҶ’Пғ$ before the activation), or for
    example a $\operatorname{softplus}$ activation function.
~~~
  - The normal distribution is used, because we can sample from it efficiently,
    we can backpropagate through it and we can compute $D_\textrm{KL}$
    analytically; furthermore, if we decide to parametrize $Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)$ using
    mean and variance, the maximum entropy principle suggests we should use the
    normal distribution.
~~~
- We use a prior $P(вҶ’z) = рқ“қ(вҶ’0, вҶ’I)$.

---
# Variational AutoEncoders Training

$$-рқ“ӣ(вҶ’Оё, вҶ’ПҶ;вҒҮвҶ’x) = рқ”ј_{Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)} \big[-\log P_{вҶ’Оё}(вҶ’x | вҶ’z)\big] + D_\textrm{KL}\big(Q_{вҶ’ПҶ}(вҶ’z | вҶ’x) \| P(вҶ’z)\big)$$

![w=50%,h=center](vae_architecture.svgz)

Note that the loss has 2 intuitive components:
- **reconstruction loss** вҖ“ starting with $вҶ’x$, passing though $Q_{вҶ’ПҶ}$, sampling
  $вҶ’z$ and then passing through $P_{вҶ’Оё}$ should arrive back at $вҶ’x$;
~~~
- **latent loss** вҖ“ over all $вҶ’x$, the distribution of $Q_{вҶ’ПҶ}(вҶ’z | вҶ’x)$ should be as close as
  possible to the prior $P(вҶ’z) = рқ“қ(вҶ’0, вҶ’I)$, which is independent on $вҶ’x$.

---
# Variational AutoEncoders вҖ“ Reparametrization Trick

In order to backpropagate through $вҶ’zвҲјQ_{вҶ’ПҶ}(вҶ’z | вҶ’x)$, note that if
$$вҶ’z вҲј рқ“қ(вҶ’Ој, вҶ’Пғ^2),$$

~~~
we can write $вҶ’z$ as
$$вҶ’z вҲј вҶ’Ој + вҶ’Пғ вҠҷ рқ“қ(вҶ’0, вҶ’I).$$

~~~
Such formulation then allows differentiating $вҶ’z$ with respect to
$вҶ’Ој$ and $вҶ’Пғ$ and is called a **reparametrization trick** (Kingma and Welling, 2013).

# Generative Models

![w=46%,f=right](generative_models.png)

There are several approaches how to represent a probability distribution
$P(вҒҮвҶ’x)$.
~~~
**Likelihood-based models** represent the probability density function directly,
often using an unnormalized probabilistic model (also called energy-based model;
i.e., specifying a non-zero _score_ or _density_ or _logits_):
$$P_{вҶ’Оё}(вҒҮвҶ’x) = \frac{e^{f_{вҶ’Оё}(вҒҮвҶ’x)}}{Z_{вҶ’Оё}}.$$

~~~
However, estimating the normalization constant $Z_{вҶ’Оё} = вҲ«e^{f_{вҶ’Оё}(вҒҮвҶ’x)}\dвҒҮвҶ’x$ is
often intractable.

~~~
- We can compute $Z_{вҶ’Оё}$ by restricting the model architecture (sequence
  modeling, invertible networks in normalizing flows);
~~~
- we can only approximate it (using for example variational inference as in
  VAE);
~~~
- we can use **implicit generative models**, which avoid representing likelihood
  (like GANs).

# Generative Adversarial Networks

We have a **generator** $G(вҶ’z; вҶ’Оё_g)$, which given $вҶ’z вҲј P(вҒҮвҶ’z)$ generates data $вҶ’x$.

~~~
Then we have a **discriminator** $D(вҶ’x; вҶ’Оё_d)$, which given data $вҶ’x$ generates a probability
whether $вҶ’x$ comes from real data or is generated by a generator.

~~~
The discriminator and generator play the following game:
$$\min_G \max_D рқ”ј_{вҶ’x вҲј P_\textrm{data}}[\log D(вҶ’x)] + рқ”ј_{вҶ’z вҲј P(вҒҮвҶ’z)}[\log (1 - D(G(вҶ’z)))].$$

![w=48%,h=center](gan_architecture.png)

---
# Generative Adversarial Networks

![w=75%,h=center](gan_training.svgz)

The generator and discriminator are alternately trained, the discriminator by
$$\argmax_{вҶ’Оё_d} рқ”ј_{вҶ’x вҲј P_\textrm{data}}[\log D(вҶ’x)] + рқ”ј_{вҶ’z вҲј P(вҒҮвҶ’z)}[\log (1 - D(G(вҶ’z)))]$$
and the generator by
$$\argmin_{вҶ’Оё_g} рқ”ј_{вҶ’z вҲј P(вҒҮвҶ’z)}[\log (1 - D(G(вҶ’z)))].$$

~~~
Basically, the discriminator acts as a trainable loss for the generator.

---
# Generative Adversarial Networks

Because $\log (1 - D(G(вҶ’z)))$ can saturate at the beginning of the training,
where the discriminator can easily distinguish real and generated samples,
the generator can be trained by
$$\argmin_{вҶ’Оё_g} рқ”ј_{вҶ’z вҲј P(вҒҮвҶ’z)}[-\log D(G(вҶ’z))]$$
instead, which results in the same fixed-point dynamics, but much stronger
gradients early in learning.

~~~
On top of that, if you train the generator by using вҖңrealвҖқ as the gold label
of the discriminator, you naturally get the above loss (which is the negative log
likelihood, contrary to the original formulation).

# GANs are Problematic to Train

Unfortunately, alternating SGD steps are not guaranteed to reach even
a local optimum of a minimax problem, consider the following one:
$$\min_x \max_y xвӢ…y.$$

~~~
The update rules of $x$ and $y$ for learning rate $Оұ$ are
$$\begin{bmatrix} x_{n+1} \\ y_{n+1} \end{bmatrix} = \begin{bmatrix} 1 & -Оұ \\ Оұ & 1 \end{bmatrix} \begin{bmatrix} x_n \\ y_n \end{bmatrix}.$$

~~~
The update matrix is a rotation matrix multiplied by a constant $\sqrt{1 + Оұ^2} > 1$
$$\begin{bmatrix} 1 & -Оұ \\ Оұ & 1 \end{bmatrix} = \sqrt{1 + Оұ^2} вӢ… \begin{bmatrix} \cos ПҶ & -\sin ПҶ \\ \sin ПҶ & \cos ПҶ \end{bmatrix},$$
so the SGD will not converge with arbitrarily small step size.

# Comparison of VAEs and GANs

The Variational Autoencoders:
- are theoretically-pleasing;
~~~
- also provide an encoder, so apart from generation, they can be used as
  unsupervised feature extraction (the VAE encoder is used in various
  modeling architectures);
~~~
- the generated samples tend to be blurry, especially with $L^1$ or $L^2$ loss
  (because of the sampling used in the reconstruction; patch-based discriminator
  with perceptual loss helps).

~~~
The Generative Adversarial Networks:
- offer high sample quality;
~~~
- are difficult to train and suffer from mode collapse.

~~~
In past few years, GANs saw a big development, improving the sample quality
substantially.
~~~
However, since 2019/2020, VAEs have shown remarkable progress
(alleviating the blurriness issue by using perceptual loss and a 2D grid of
latent variables), and are being used for generation too.
~~~
Furthermore, additional approaches (normalizing flows, diffusion models) were
also being explored, with diffusion models becoming the most promising approach
since Q2 of 2021, surpassing both VAEs and GANs.

# Wasserstein GAN

Standard GANs optimize the Jensen-Shannon divergence
$$\operatorname{JSD}(p \| q) = \frac{1}{2} D_\textrm{KL}\big(p \| (p+q)/2\big) + \frac{1}{2} D_\textrm{KL}\big(q \| (p+q)/2\big),$$
because for a fixed generator $G$, the optimum discriminator $D^*_G(вҶ’x) = \frac{P_\textrm{data}(вҶ’x)}{P_\textrm{data}(вҶ’x) + P_\textrm{generator}(вҶ’x)}.$

~~~
Therefore,
$$\begin{aligned}
  & рқ”ј_{вҶ’x вҲј P_\textrm{data}}[\log D^*_G(вҶ’x)] + рқ”ј_{вҶ’z вҲј P(вҒҮвҶ’z)}[\log (1 - D^*_G(G(вҶ’z)))] \\
={} & рқ”ј_{вҶ’x вҲј P_\textrm{data}}[\log D^*_G(вҶ’x)] + рқ”ј_{вҶ’x вҲј P_\textrm{generator}}[\log (1 - D^*_G(вҶ’x)))] \\
={} & рқ”ј_{вҶ’x вҲј P_\textrm{data}}\left[\log\frac{P_\textrm{data}(вҶ’x)}{P_\textrm{data}(вҶ’x) + P_\textrm{generator}(вҶ’x)}\right] + рқ”ј_{вҶ’x вҲј P_\textrm{generator}}\left[\log \frac{P_\textrm{generator}(вҶ’x)}{P_\textrm{data}(вҶ’x) + P_\textrm{generator}(вҶ’x)}\right] \\
={} & D_\textrm{KL}\left(P_\textrm{data}\middle\Vert\frac{P_\textrm{data} + P_\textrm{generator}}{2}\right) + D_\textrm{KL}\left(P_\textrm{generator}\middle\Vert\frac{P_\textrm{data} + P_\textrm{generator}}{2}\right) + c.
\end{aligned}$$

# Wasserstein GAN

Instead of minimizing JS divergence
$$\operatorname{JSD}(p \| q) = \frac{1}{2} D_\textrm{KL}\big(p \| (p+q)/2\big) + \frac{1}{2} D_\textrm{KL}\big(q \| (p+q)/2\big),$$

~~~
Wasserstein GAN minimizes the Wasserstein distance (also called the earth mover's distance):
$$W(p, q) = \inf_{Оі вҲҲ О (p, q)} рқ”ј_{(x, y) вҲј Оі} \big[ \|x - y\| \big].$$

~~~
The joint distribution $Оі вҲҲ О (p, q)$ indicates how much вҖңmassвҖқ must be transported
from $x$ to $y$, and EM is the вҖңcostвҖқ of the optimal transport plan.

# BigGAN Ingredients вҖ“ Spectral Normalization

Satisfying the Lipschitz constraint by truncation is not very effective.
Better approaches were proposed, by using for example gradient penalties
(WGAN-GP) or **spectral normalization**.

~~~
In spectral normalization, the idea is to keep the **spectral norm** (the largest
singular value) of all convolutional and dense layers equal or close to 1,
which in turn guarantees the Lipschitz constraint of the model.

~~~
![w=70,f=right](spectral_normalization.svgz)

Spectral normalization can be implemented efficiently by performing one step of
power iteration each time the kernel in question is used in training.
