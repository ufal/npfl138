# What are Neural Networks

Neural networks are just a model for describing computation of outputs from
given inputs.

~~~
#
The model:
- is strong enough to approximate any reasonable function,

~~~
- is reasonably compact,
~~~
- allows heavy parallelization during execution (GPUs, TPUs, ‚Ä¶).

~~~
#
Nearly all the time, neural networks generate a _probability distribution_ as
output:

~~~
- distributions allow small changes during training,

~~~
- during prediction, we usually take the most probable outcome (class/label/‚Ä¶).

~~~
#
When there is enough data, neural networks are currently the best performing
machine learning model, especially when the data are high-dimensional (images,
videos, speech, texts, ‚Ä¶).

# Linear Algebra Conventions & Broadcasting

## Vector Addition and Other Element-Wise Operations

Classic linear algebra distinguishes between **column vectors** ($N\times 1$
matrices) and **row vectors** ($1\times N$ matrices), often treating
the addition of a row and a column as undefined.

~~~
In the context of **deep learning**, however, vectors outside of matrix
multiplication are treated simply as **1D arrays**. Therefore, adding two
vectors is valid as long as their lengths match, even if one originates as a row
and the other as a column.

~~~
## Matrix-Vector Broadcasting

![w=49%,f=right](broadcasting.png)

When adding a matrix and a vector, we adopt **broadcasting conventions** common in
NumPy and PyTorch.

~~~
The vector is interpreted as a matrix with a single row, and this
row is then **implicitly copied** (broadcasted) to match the shape of the
matrix.

# Random Variables

A random variable $‚Āáx$ is a result of a random process, and it can be either
discrete or continuous.

~~~
## Probability Distribution
A probability distribution describes the likelihood of each possible value
a random variable can take.

The notation $‚Āáx ‚ąľ P$ stands for a random variable $‚Āáx$ having a distribution $P$.

~~~
For discrete variables, the probability that $‚Āáx$ takes a value $x$ is denoted as
$P(x)$ or explicitly as $P(‚Āáx = x)$. All probabilities are nonnegative, and the
sum of the probabilities of all possible values of $‚Āáx$ is $‚ąĎ_x P(‚Āáx=x) = 1$.

~~~
For continuous variables, the probability that the value of $‚Āáx$ lies in the interval
$[a, b]$ is given by $‚ąę_a^b p(x)\d x$, where $p(x)$ is the _probability density
function_, which is always nonnegative and integrates to 1 over the range of
all values of $‚Āáx$.

---
style: .katex-display { margin: .8em 0 }
# Joint, Conditional, Marginal Probability

![w=44%,f=right](joint_probability.svgz)

For two random variables, a **joint probability distribution** is a distribution
of all possible pairs of outputs (and analogously for more than two):

$$P(‚Āáx = x_2, ‚Āáy = y_1).$$

~~~
**Marginal distribution** is a distribution of one (or¬†a¬†subset) of the random
variables and can be obtained by summing over the other variable(s):
$$P(‚Āáx=x_2) = {\small ‚ąĎ\nolimits}_y P(‚Āáx = x_2, ‚Āáy = y).$$

~~~
**Conditional distribution** is a distribution of one (or a subset) of the
random variables, given that another event has already occurred:
$$P(‚Āáx=x_2 \mid ‚Āáy=y_1) = P(‚Āáx = x_2, ‚Āáy = y_1) / P(‚Āáy = y_1).$$

~~~
If $P(‚Āáx\!=\!x, ‚Āáy\!=\!y) = P(‚Āáx\!=\!x)\!‚čÖ\!P(‚Āáy\!=\!y)$ for all $x,y$, random
variables $‚Āáx, ‚Āáy$ are **independent**.

# Random Variables

## Expectation
The expectation of a function $f(x)$ with respect to a discrete probability
distribution $P(‚Āáx)$ is defined as:
$$ūĚĒľ_{‚Āáx ‚ąľ P}[f(x)] ‚ČĚ ‚ąĎ_x P(x)f(x).$$

~~~
For continuous variables, the expectation is computed as:
$$ūĚĒľ_{‚Āáx ‚ąľ p}[f(x)] ‚ČĚ ‚ąę_x p(x)f(x)\d x.$$

~~~
If the random variable is obvious from context, we can write only $ūĚĒľ_P[x]$,
$ūĚĒľ_{‚Āáx}[x]$, or even $ūĚĒľ[x]$.

~~~
Expectation is linear, i.e., for constants $őĪ, ő≤ ‚ąą ‚ĄĚ$:
$$ūĚĒľ_{‚Āáx} [őĪf(x) + ő≤g(x)] = őĪūĚĒľ_{‚Āáx} [f(x)] + ő≤ūĚĒľ_{‚Āáx} [g(x)].$$

---
# Random Variables

## Variance
Variance measures how much the values of a random variable differ from its
mean $ūĚĒľ[x]$.

$$\begin{aligned}
  \Var(x) &‚ČĚ ūĚĒľ\left[\big(x - ūĚĒľ[x]\big)^2\right]\textrm{, or more generally,} \\
  \Var_{‚Āáx ‚ąľ P}(f(x)) &‚ČĚ ūĚĒľ\left[\big(f(x) - ūĚĒľ[f(x)]\big)^2\right].
\end{aligned}$$

~~~
It is easy to see that
$$\Var(x) = ūĚĒľ\left[x^2 - 2x‚čÖūĚĒľ[x] + \big(ūĚĒľ[x]\big)^2\right] = ūĚĒľ\left[x^2\right] - \big(ūĚĒľ[x]\big)^2,$$
because $ūĚĒľ\big[2x‚čÖūĚĒľ[x]\big] = 2(ūĚĒľ[x])^2$.

~~~
Variance is connected to $ūĚĒľ[x^2]$, the **second moment** of a random
variable ‚Äď it is in fact a **centered** second moment.

# Common Probability Distributions
## Bernoulli Distribution
The Bernoulli distribution is a distribution over a binary random variable.
It has a single parameter $ŌÜ ‚ąą [0, 1]$, which specifies the probability that
the random variable is equal to 1.

~~~
$$\begin{aligned}
  P(x) &= ŌÜ^x (1-ŌÜ)^{1-x} \\
  ūĚĒľ[x] &= ŌÜ \\
  \Var(x) &= ŌÜ(1-ŌÜ)
\end{aligned}$$

![w=60%,h=center](bernoulli_variance.svgz)

# Common Probability Distributions

## Categorical Distribution
Extension of the Bernoulli distribution to random variables taking one of $K$ different
discrete outcomes. It is parametrized by $‚Üíp ‚ąą [0, 1]^K$ such that $‚ąĎ_{i=0}^{K-1} p_{i} = 1$.

~~~
We represent outcomes as vectors $‚ąą \{0, 1\}^K$ in the **one-hot encoding**.
Therefore, an outcome $x ‚ąą \{0, 1, ‚Ä¶, K-1\}$ is represented as a vector
$$‚Üí1_x ‚ČĚ \big([i = x]\big)_{i=0}^{K-1} = \big(\underbrace{0, ‚Ä¶, 0}_{x}, 1, \underbrace{0, ‚Ä¶, 0}_{K-x-1}\big).$$

~~~
The outcome probability, mean, and variance are very similar to the Bernoulli
distribution.
$$\begin{aligned}
  P(‚Üíx) &= ‚ąŹ\nolimits_{i=0}^{K-1} p_i^{x_i} \\
  ūĚĒľ[x_i] &= p_i \\
  \Var(x_i) &= p_i(1-p_i) \\
\end{aligned}$$

# Information Theory

## Self-Information

Self-information can be considered the amount of **surprise** when a random variable is sampled.
~~~
- Should be zero for events with probability 1.
~~~
- Less likely events are more surprising.
~~~
- Independent events should have **additive** surprise (information).

~~~
![w=26.5%,f=right](self_information_plot.svgz)

These conditions are fulfilled by **self-information** $I(x)$, also called **surprise**:
$$I(x) ‚ČĚ -\log P(x) = \log \frac{1}{P(x)}.$$

---
# Information Theory

## Entropy

Amount of **surprise** in the whole distribution.
$$H(P) ‚ČĚ ūĚĒľ_{‚Āáx‚ąľP}[I(x)] = -ūĚĒľ_{‚Āáx‚ąľP}[\log P(x)]$$

~~~
- for discrete $P$: $H(P) = -‚ąĎ_x P(x) \log P(x)$
- for continuous $P$: $H(P) = -‚ąę P(x) \log P(x)\,\mathrm dx$

~~~
Because $\lim_{x ‚Üí 0} x \log x = 0$, for $P(x) = 0$ we consider\
$P(x) \log P(x)$ to be zero.

~~~ ~~~
![w=40%,f=right](entropy_example.svgz)

- for discrete $P$: $H(P) = -‚ąĎ_x P(x) \log P(x)$
- for continuous $P$: $H(P) = -‚ąę P(x) \log P(x)\,\mathrm dx$

Because $\lim_{x ‚Üí 0} x \log x = 0$, for $P(x) = 0$ we consider
$P(x) \log P(x)$ to be zero.

~~~
Note that in the continuous case, the continuous entropy (also called
_differential entropy_) has slightly different semantics, for example, it can be
negative.

~~~
For binary logarithms, the entropy is measured in **bits**. However,
from now on, all logarithms are _natural logarithms_ with base _e_
(and then the entropy is measured in units called **nats**).

---
# Information Theory

## Cross-Entropy

$$H(P, Q) ‚ČĚ -ūĚĒľ_{‚Āáx‚ąľP}[\log Q(x)]$$

~~~
**Gibbs inequality** states that
- $H(P, Q) ‚Č• H(P)$
- $H(P) = H(P, Q) ‚áĒ P = Q$
~~~
- Proof: Using the fact that $\log x ‚Č§ (x-1)$ with equality only for $x=1$, we get
  $$‚ąĎ_x P(x) \log \frac{Q(x)}{P(x)} ‚Č§ ‚ąĎ_x P(x) \left(\frac{Q(x)}{P(x)}-1\right) = ‚ąĎ_x Q(x) - ‚ąĎ_x P(x) = 0.$$
~~~
- Corollary: For a categorical distribution with $n$ outcomes, $H(P) ‚Č§ \log n$,
  because for $Q(x) = 1/n$ we get $H(P) ‚Č§ H(P, Q) = -‚ąĎ_x P(x) \log Q(x) = \log n.$
~~~

Note that generally $H(P, Q) ‚Č† H(Q, P)$.

---
# Information Theory

## Kullback‚ÄďLeibler Divergence (KL Divergence)

Sometimes also called **relative entropy**.

$$D_\textrm{KL}(P \| Q) ‚ČĚ H(P, Q) - H(P) = ūĚĒľ_{‚Āáx‚ąľP}[\log P(x) - \log Q(x)]$$

~~~
- Consequence of Gibbs inequality: $D_\textrm{KL}(P \| Q) ‚Č• 0$, $D_\textrm{KL}(P \| Q) = 0$ iff $P = Q$.

~~~
- Generally $D_\textrm{KL}(P \| Q) ‚Č† D_\textrm{KL}(Q \| P)$.

# Common Probability Distributions
## Normal (or Gaussian) Distribution
Distribution over real numbers, parametrized by a mean $őľ$ and variance $ŌÉ^2$:
$$ūĚďĚ(x; őľ, ŌÉ^2) = \sqrt{\frac{1}{2ŌÄŌÉ^2}} \exp \left(-\frac{(x - őľ)^2}{2ŌÉ^2}\right)$$

~~~

![w=45%,h=center](normal_distribution.svgz)

---
# Why Normal Distribution

## Central Limit Theorem
The sum of independent identically distributed random variables
with finite variance converges to normal distribution.

~~~
## Principle of Maximum Entropy
Given a set of constraints, a distribution with maximal entropy fulfilling the
constraints can be considered the most general one, containing as little
additional assumptions as possible.

~~~
Considering distributions on all real numbers with a given mean and variance, it
can be proven (using variational inference) that such a distribution with
**maximum entropy** is exactly the normal distribution.

>  A computer program is said to learn from experience E with respect to some

# Perceptron ‚Äď Extra Simple Neural Network

![w=30%,f=right](perceptron.svgz)

- Assume we have an input node for every input feature.

~~~
- Additionally, we have an output node for every model output.

~~~
- Each input node is connected to every output node by a directed edge, and
  every edge has an associated weight.

~~~
- Value of every (output) node is computed by summing the values of predecessors
  multiplied by the corresponding weights, added to a bias of this node, and
  finally passed through an activation function $a$:
  $$y = a\left(‚ąĎ\nolimits_j x_j w_j + b\right)$$
  or in vector form $y = a(‚Üíx^\T‚Üíw+b)$, or for a batch of examples $‚áČX$,
  $‚Üíy = a(‚áČX ‚Üíw + b)$.

---
# Perceptron ‚Äď Separating Hyperplane

To classify an example in two classes, we compare the output $y = a(‚Üíx^\T‚Üíw+b)$
with some threshold $z$, and choose the resulting class depending on
whether $y > z$ or $y < z$.

~~~
When the activation function is monotonous, this is equal to comparing
$‚Üíx^\T‚Üíw+b > a^{-1}(z)$.

~~~
The resulting classes are therefore divided by a _separating hyperplane_
$‚Üíx^\T‚Üíw+c$.

# Neural Network Architecture

![w=35%,f=right](neural_network.svgz)

The computation is performed analogously to the perceptron:
$$\begin{aligned}
  h_i &= f\left(‚ąĎ\nolimits_j x_j w^{(h)}_{j,i} + b^{(h)}_i\right), \\
  y_i &= a\left(‚ąĎ\nolimits_j h_j w^{(y)}_{j,i} + b^{(y)}_i\right),
\end{aligned}$$
or in matrix form
$$\begin{aligned}
  ‚Üíh &= f\Big(‚Üíx^\T ‚áČW^{(h)} + ‚Üíb^{(h)}\Big), \\
  ‚Üíy &= a\Big(‚Üíh^\T ‚áČW^{(y)} + ‚Üíb^{(y)}\Big),
\end{aligned}$$
or for a whole batch of inputs $‚áČH = f\Big(‚áČX ‚áČW^{(h)} + ‚Üíb^{(h)}\Big)$ and $‚áČY = a\Big(‚áČH ‚áČW^{(y)} + ‚Üíb^{(y)}\Big)$.

The $‚áČW^{(h)} ‚ąą ‚ĄĚ^{|\mathit{input}|‚čÖ|\mathit{hidden}|}$ is a matrix of weights
and $‚Üíb^{(h)} ‚ąą ‚ĄĚ^{|\mathit{hidden}|}$ is a vector of biases of the first layer;
$‚áČW^{(y)} ‚ąą ‚ĄĚ^{|\mathit{hidden}|‚čÖ|\mathit{output}|}$ and $‚Üíb^{(y)} ‚ąą ‚ĄĚ^{|\mathit{output}|}$
are parameters of the second layer.

# Universal Approximation Theorem '89

Let $ŌÜ(x) \colon ‚ĄĚ ‚Üí ‚ĄĚ$ be a nonconstant, bounded and nondecreasing continuous function. \
(Later a proof was given also for $ŌÜ = \ReLU$ and even for any nonpolynomial
function.)

For any $őĶ > 0$ and any continuous function $f \colon [0, 1]^D ‚Üí ‚ĄĚ$, there exists
$H ‚ąą ‚Ąē$, $‚Üív ‚ąą ‚ĄĚ^H$, $‚Üíb ‚ąą ‚ĄĚ^H$ and $‚áČW ‚ąą ‚ĄĚ^{D√óH}$, such that if we denote
$$F(‚Üíx) = ‚Üív^\T ŌÜ(‚Üíx^\T ‚áČW + ‚Üíb) = ‚ąĎ_{i=1}^H v_i ŌÜ(‚Üíx^\T ‚ÜíW_{*, i} + b_i),$$
where $ŌÜ$ is applied element-wise,
~~~
then for all $‚Üíx ‚ąą [0, 1]^D$:
$$|F(‚Üíx) - f(‚Üíx)| < őĶ.$$

~~~
## One Possible Interpretation

It is always possible to create features using just a single linear layer
followed by a nonlinearity, such that the resulting dataset is always linearly
separable.

# Universal Approximation Theorem for ReLUs

# Summary

Neural networks are just a model for describing computation of outputs from
given inputs:
- strong enough to approximate any reasonable function,
- reasonably compact,
- allows heavy parallelization during execution (GPUs, TPUs, ‚Ä¶).

Nearly all the time, neural networks generate a _probability distribution_ on
output:
- output distribution determines the output activation function,
- distributions allow small changes during training,
- during prediction, we usually take the most probable outcome (class/label/‚Ä¶).

When there is enough data, neural networks are currently the best performing
machine learning model, especially when the data are high-dimensional (images,
speech, text).
- For tabular data, you should also consider gradient-boosted decision trees.
